# bicycle_race — throwaway experiment (NOT the game)

Минимальная «F1-гонка» на движке [PathTrackingBicycle](https://github.com/DongChen06/PathTrackingBicycle):
кинематическая велосипедная модель + Stanley (руль) + PID (скорость), каждая машина
едет по одной гоночной линии (`racetrack_waypoints.txt` — несёт целевую скорость в каждой
точке, в поворотах ниже). N машин с разным «скиллом» (множитель скорости) спринтуют от
разнесённой решётки до финиша.

**Это эксперимент чисто пощупать позиционный движок** — противоположность абстрактному
lap-time симу игры. К `ApexDuo_Prototype/` и `ApexWeb/` отношения не имеет, изолировано.

## Запуск
```
# реальная трасса из SVG (замкнутая, круги) — рекомендуется:
python experiments/bicycle_race/race.py --svg svg/monza-7.svg --laps 3
python experiments/bicycle_race/race.py --svg svg/suzuka-2.svg --laps 2 --cars 8
python experiments/bicycle_race/race.py --svg svg/catalunya-6.svg --headless   # без рендера
# исходная открытая трасса репо (спринт точка-в-точку):
python experiments/bicycle_race/race.py
```
Нужны: numpy, matplotlib. SVG-трассы в `svg/` (f1-circuits-svg, ROY Jules, CC BY 4.0).

## Реальные трассы из SVG (добавлено)
`--svg <файл>` грузит контур реального автодрома через парсер игры
(`tools/svg_to_trackshapes.py`): замкнутый контур → **круги** (`--laps`), а скорость в
каждой точке выводится из **кривизны** (в поворотах медленнее) с торможением ПЕРЕД
поворотом (длинная зона торможения, чтобы машина доезжала до апекса уже медленной).
Проверено на Монце/Каталунье/Сузуке: 6 машин, 2 круга, все финишируют, 8–9 обгонов,
наложений нет.

## Границы трассы и повороты (добавлено)
- **Ширина/края:** из центральной линии строятся левый/правый край (смещение по нормали
  на полширины `TRACK_WIDTH`), трасса рисуется лентой. Ширина подобрана так, чтобы
  траектория контроллера + «свес» в поворотах помещались внутри (лимит срабатывает <0.5%).
- **Удержание в границах:** если машину вынесло за кромку — её мягко возвращают на край
  (track limits). Машины не вылетают за пределы.
- **Повороты:** короткое окно обзора контроллера (`WINDOW=10`) держит линию плотно (не
  срезает), угол руля и Stanley настроены под прохождение реальных шикан.
- **Обгон с противоположной стороны:** атакующий уходит на сторону, ПРОТИВОПОЛОЖНУЮ машине
  впереди (иначе все валятся в одну полосу и встают в дедлок).

## Что показал тест
Трасса не замкнута (старт и финиш в 838 м) → спринт точка-в-точку, а не круги.
6 машин проходят трассу целиком (~1723/1724 вейпойнта). Быстрые машины стартуют С ХВОСТА
и пробиваются вперёд: VER (старт P6, скилл 1.09) → финиш P2, медленный HAM (P1) → P6,
суммарно ~9 позиций отыграно. `race.gif` — превью.

## Машины по размеру (добавлено)
Рисуются и сталкиваются как ориентированные прямоугольники `CAR_LENGTH × CAR_WIDTH` (20×9
ед.). Размер преувеличен ради видимости — настоящие 5×2 м на трассе ~1756 ед. были бы точкой;
зато физика и рендер используют ОДИН размер (согласованно).

## Обгоны + защита от столкновений (добавлено)
- **Следование по полосе:** машина в «той же полосе» (центры ближе `LAT_CONFLICT` вбок) не
  может приблизиться к переднему ближе `MIN_SEP` — скорость плавно подводится к скорости
  впереди (car-following). Проехать сквозь нельзя.
- **Обгон = уйти из полосы:** чтобы пройти, нужно сместить отслеживаемую линию вбок (`OFFSET`,
  Stanley ведёт по параллельной траектории) — тогда полоса очищается и ограничение скорости
  снимается. Прошёл — вернулся на линию.
- **Жёсткая страховка:** если два болида одной полосы всё же пересеклись, их растаскивает по
  линии центров. Проверено headless: минимальная дистанция в одной полосе ≈ длине машины → НЕТ
  наложений; быстрые с хвоста реально пробиваются вперёд (VER P6→P2).

## Чего тут НЕТ (и почему это не для игры)
Настоящей борьбы колесо-в-колесо, выбора стороны/защитных линий, детерминизма под netcode,
аэро/грязного воздуха. Это причины, по которым основной игре позиционный движок не подходит
(см. ApexWeb/TODO.md, блок «не брать», и план «зон обгона» п.2b — текстура без геометрии).
Здесь — только пощупать, как ведёт себя позиционный движок.
