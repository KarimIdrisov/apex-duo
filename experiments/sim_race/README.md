# sim_race — PoC spike (NOT the game)

Доказательство одной идеи: **позиционную гонку «как в `bicycle_race`» можно
рисовать поверх абстрактного lap-time движка** — без велосипедной модели, без
Stanley/PID.

Тут нет физики руления. Положение машины на трассе берётся из одного скаляра
`progress = lap + lap_frac`, который каждый тик (`DT = 0.25s`, time-discrete)
двигает **абстрактная модель времени круга**:
- скилл пилота → базовое время круга;
- компаунд + износ шин (линейный + «обрыв»);
- топливо (тяжёлая машина едет медленнее, разгружается за стинт);
- один плановый пит-стоп (потеря времени, смена компаунда);
- борьба колесо-в-колесо: грязный воздух + накопление «pass credit» по дельте
  темпа, завершённый обгон реально переставляет машину вперёд.

**Профиль скорости (фил F1).** Чтобы выглядело по-гоночному, машины не едут по
кругу равномерно: `speed_profile()` считает скорость в каждой точке трассы —
friction-limited в поворотах (`v = sqrt(A_LAT / кривизна)`, потолок `VMAX_U`) +
проход торможения и проход разгона (g-g идея, **позаимствована у fastest-lap /
laptime-simulation, код свой**). `lap_frac` двигается пропорционально локальной
скорости (медленно в поворотах, быстро на прямых), с нормировкой — суммарное
время круга остаётся ровно `clean_laptime()`. В анимации осевая линия раскрашена
по скорости: зелёный = быстро, красный = тормозная зона.

**Траектория решает.** У каждого пилота навык линии `line` (0..1). Лучше линия →
ниже время круга (`LINE_LAP_GAIN`). Когда машину сдувает с идеальной линии (обгон
по внешней / защита, `offset != 0`), она теряет скорость в повороте
(`OFFLINE_CORNER_COST`) — отсюда естественные свичбэки (обошёл по внешней →
проиграл на выходе).

**Коллизии / инциденты.** Обгон — это попытка с риском (`_attempt_pass`): риск
выше в медленных поворотах (`INCIDENT_CORNER_MULT`) и у пилотов с плохой линией.
Исходы: чисто прошёл; лёгкий контакт (обгон сорван, потеря времени); разворот
(большая потеря); редко — сход (DNF, машина гаснет на трассе). Плюс
геометрическая страховка `step_field`: машины одной полосы не налезают друг на
друга (держится `MIN_GAP`). Всё детерминировано от `--seed`.

Рендер (контур трассы из SVG, ориентированные прямоугольники-машинки, таблица с
гэпами) переиспользован из `bicycle_race`. `lap_frac → точка на осевой линии`
по равномерным точкам `resample_closed`; боковое смещение при обгоне —
косметика.

Это набросок «подхода C» для будущего менеджмент-движка: time-discrete рантайм
даёт **и** достоверные числа (позиции/гэпы/стратегия), **и** нативно плавную
позиционную анимацию. Места, где сейчас стоят эвристики (`COMPOUNDS`,
`FUEL_PER_LAP`, `BASE_LAP`), — это куда позже запекаются физичные времена из
TUMFTM `laptime-simulation`.

## Запуск
```
python experiments/sim_race/race.py                                  # живая анимация (Монца)
python experiments/sim_race/race.py --svg svg/suzuka-2.svg --laps 20
python experiments/sim_race/race.py --cars 8 --seed 4                 # другая гонка (есть DNF)
python experiments/sim_race/race.py --headless                       # без рендера, печать итога
python experiments/sim_race/race.py --save out.gif --spf 1.0         # сохранить GIF
```
Нужны: numpy, matplotlib (+ Pillow для `--save`). Трассы берутся из
`../bicycle_race/svg/`.

Скорость показа:
- `--spf` — сколько СЕКУНД гонки проходит за один кадр (меньше = медленнее/плавнее, по умолч. 1.0);
- `--fps` — скорость воспроизведения GIF (по умолч. 30).

## Что показал прогон (Монца, 8 машин, 12 кругов, `--seed 4`)
Быстрые стартуют с хвоста. HAM (P6) прорезал до P1 (+5, 5 обгонов), ALO (P7) → P2,
поле отсортировалось по темпу через борьбу; самый быстрый SAI (P8) после трёх
контактов **сошёл (DNF)**. Превью — `sim_race_monza.gif`. На дистанции ~4–5
инцидентов и ~1 сход за гонку.

## Чего тут НЕТ (это спайк, не движок)
Детерминизма под netcode (numpy RNG, не seeded-LCG), реальной калибровки времён,
SC/погоды, нескольких питов и тайр-инвентаря, реальной геометрии контакта (контакт
разыгрывается в момент попытки обгона, не по столкновению боксов). Цель — пощупать
связку «абстрактный движок → живая позиционная F1-картинка» с траекторией и
инцидентами.
