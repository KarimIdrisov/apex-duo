# Apex Duo — Дизайн-спецификация интерфейса (UI / Visual Spec) v1.1

Документ фиксирует визуальный язык игры, чтобы дизайнер мог по нему работать.
**Север-звезда: графика реальной ТВ-трансляции Formula 1** (timing tower, плашки
команд, «шашка», секторные цвета) — наложенная на USP игры: **кооператив на одну
команду** с двумя ролями, **P5 — Директор** и **P6 — Инженер**.

> **v1.1 — строгий проход.** По фидбэку дизайн сделан строже и плотнее: мельче
> типографика, веса не выше **700** (900 убран), **моноширинные цифры**, тоньше
> линии, приглушённая графитовая база, цвета команд/ролей — мелкими акцентами
> (узкие плашки/точки/слабые тинты ≈7%), **без крупных скосов и ярких заливок**.
> Тон — премиальный «инженерный / timing-software», а не аркадный.

> **Важно.** Этот документ фиксирует *дизайн*, а не код. Ни один `.gd`-файл здесь не
> меняется. Перенос значений в код (`main.gd`, `season_hub.gd`, `track_map.gd`,
> `f1_2026.gd`) — отдельная задача реализации; см. §12 «Маппинг на код».

**Компаньоны к этому документу**

- `docs/mockups/apex_duo_ui_mockups.html` — кликабельные макеты всех 5 экранов
  (открыть в браузере; переключение вкладками сверху). Версия v1.1.
- `docs/RACE_VISUAL_SPEC.md` — детальная спека мини-карты/маркеров (этот документ
  с ней согласован и расширяет её до полной системы).

## 1. Арт-дирекшн

**Что копируем у реальной F1**

- **Timing tower** слева: позиция → плашка-цвет команды → имя → отрыв → шина →
  статус (PIT/OT/CLIP). Лидер выделен, быстрейший — фиолетовым.
- **Секторные цвета** трансляции: фиолетовый = лучший в сессии, зелёный = личный
  лучший, жёлтый = медленнее.
- **Шины Pirelli** как круглые цветные «таблетки» с буквой.
- Плотность данных как в трансляции, но **сгруппированная по смыслу**, без шума.

**Принципы**

1. **Графитовая база + один красный акцент.** Красный — только тонкие линии, рамки
   выбора и CTA. Не красить им блоки.
2. **Цвет = смысл.** Команда, шина, заряд, состояние — у каждого свой устойчивый
   цвет, поданный *мелко*.
3. **Кооп виден везде.** Золотой (P5) и голубой (P6) — сквозной язык: в таблице, на
   карте, в панелях, в зачёте. Но **сдержанно**: узкий стрип + слабый тинт.
4. **Colour-blind-safe.** Любое ключевое отличие закодировано **и формой, и цветом**
   (диск/кольцо, буква на шине, иконка, текст на чипе) — никогда только цветом.
5. **Сдержанность вместо «игровухи».** Никаких крупных скосов, ярких заливок и
   тяжёлых весов — hairline-линии, плоские шапки, моноцифры. Акцент работает,
   потому что вокруг тихо.

### 2.4 Кооп-идентичность (USP)

| Токен | HEX | Роль |
|---|---|---|
| `--p5` | `#FFD166` | **P5 — Директор** (золото) |
| `--p6` | `#66C2FF` | **P6 — Инженер** (голубой) |

**Правило применения.** P5/P6 — это *идентичность игрока*, накладываемая **поверх**
цвета команды, а не вместо него. На карте: цвет команды = заливка маркера, золото/
голубой = гало-кольцо вокруг. В таблице: цвет команды = плашка-tab слева, золото/
голубой = тонкий стрип у самого края + подсветка фона строки.

**Сдержанно.** Подсветка строки — слабый тинт (**≈7% альфа**), стрип **2px**, не
яркий градиент. Цвета `--p5/--p6` оставляем каноничными (как в коде), но применяем
тихо — идентичность читается, но не «кричит».

## 3. Типографика

- **Текст/заголовки:** `Formula1` (лиценз.) → стенд-ин **`Titillium Web`**.
  **Цифры** (позиции, отрывы, времена, %, деньги) — моноширинный **`Roboto Mono`**:
  это сразу даёт «инженерный / timing-software» тон.
- **Веса:** только **400 / 600 / 700**. **900 не используем** — именно он делал
  картинку «мультяшной». Заголовки — 700, акцент — 600.
- **Табличные цифры обязательны** (`tabular-nums`) — колонки не «прыгают».
- **Регистр:** мелкие моно-ярлыки ВЕРХНИМ регистром с трекингом `.1em` (как в
  телеметрии). Имена пилотов и описания — обычным регистром.

| Роль | Размер / вес | Шрифт |
|---|---|---|
| Титул меню | 44–46 / 700 | Titillium |
| H1 экрана | 18 / 700, ВЕРХ | Titillium |
| Лап-счётчик / позиция / отрыв | 12–18 / 500 | Roboto Mono |
| Шапка панели (ярлык) | 10 / 500, ВЕРХ `.12em` | Roboto Mono |
| Имя в таблице | 12–13 / 600 | Titillium |
| Body / описание | 13–14 / 400 | Titillium |
| Микро-ярлык | 9.5–10 / 500, ВЕРХ | Roboto Mono |

## 4. Сетка, отступы, форма

- **База отступов 8px** (шаг 4 / 8 / 12 / 16 / 24), но **плотно**: внутренние
  паддинги панелей 8–11px, строки тайминга **25px**, не больше.
- **Радиусы:** 4px (панели, кнопки), 3px (чипы, сегменты), 2px (бары). Плашки и
  «шашка» — без радиуса.
- **Границы:** hairline `1px var(--line)`; мягкая зебра строк через `--row-alt`.
- **Без скосов.** Шапка — **плоский бар** (44px) с тонкой красной линией снизу
  (2px) как единственным «брендом». Никаких параллелограммов.
- **Каркас экрана:** `плоская шапка` → `контент` → `нижний бар`. Гоночный HUD
  внутри — 3 колонки (см. §7.4).

## 5. Библиотека компонентов

### 5.1 Шапка (flat header)
**Плоский** бар высотой 44px: лого `APEX DUO` (DUO красным), тонкий разделитель,
название+тип трассы; справа — лап-счётчик `14 / 44` (моно) и статус-пилюля
(ГОНКА/ПАУЗА/SC/ФИНИШ) **тонкой обводкой**, не заливкой. Снизу — красная линия 2px.

### 5.2 Кнопки
- **CTA:** прозрачная, рамка `--red`, текст красный 700. Главное действие экрана.
- **Стандартная:** `--panel-2`, граница `--line`, текст `--text`, мелкая. Активная
  (×2, пауза) — текст/рамка `--p6`.
- **Ghost / меню:** прозрачная, левый цветной стрип **2px** (роль: красный=сезон,
  зелёный=продолжить, нейтр.=прочее).

### 5.3 Сегментированный контрол (pace / ERS)
Группа из 3 кнопок в общей рамке с hairline-разделителями. Невыбранные — прозрачные,
текст `--dim`. Выбранная — **не яркая заливка, а подчёркивание**: лёгкий фон
`--panel-2`, текст цветом роли + **нижняя граница 2px** этим же цветом:
- темп **P5** → `--p5`, темп **P6** → `--p6`, **ERS** → `--ers`, **общий темп** →
  `--text` (нейтральный, «обоим»). Контрол читается, но не выглядит «игровым».

### 5.4 Строка timing tower (анатомия)
`[ident 2px][POS 26 моно][tab-команда 3][ИМЯ …][ОТРЫВ 54 моно][шина 16][статус]`,
высота **25px**, мягкая зебра.
- Лидер: POS светлым, «корона» ▲ слева, отрыв фиолетовым.
- Твои пилоты: слабый тинт строки (≈7%) + ident-стрип у края + имя цветом P5/P6.
- Статус-чип: PIT (синий), OT (янтарь), CLIP (стальной) — **обводка**, не заливка.

### 5.5 Tyre chip
Круг Ø16, заливка компаунда, **буква** S/M/H/I/W (моно, тёмная/светлая по фону),
контур 1px. Буква обязательна (colour-blind).

### 5.6 Батарея / ERS gauge
Сегментированная шкала (≈14 тонких ячеек), высота 9px. Заполнение по заряду:
`>60% --ers`, `25–60% --warn`, `<25% --bad`. **Клиппинг:** ячейки → `--clip` +
иконка-молния «пусто». Число рядом — моно, цветом состояния.

### 5.7 Wear bar (износ)
Тонкая полоса (6px) с заливкой `--sector-green / --sector-yellow / --bad`
(порог 50 / 78%). Число справа — моно, тем же цветом.

### 5.8 Кокпит-карта пилота (кооп)
Контейнер с тонким левым бордюром (P5 золото / P6 голубой) и шапкой
`P5 · Норрис … ДИРЕКТОР`. Внутри: шина+износ, режим темпа (сегмент), батарея,
режим ERS (сегмент), кнопка «⚡ Обгон» (обводка/заливка по состоянию), пит-стоп.
**Две карты — зеркальные**: одинаковая раскладка, разный цвет идентичности.

### 5.9 Бейджи / пилюли состояний
`SC` (жёлтый), `PIT`/`IN`/`OUT` (синий), `OT` (янтарь), `FASTEST` (фиолетовый),
`OUT` (серый, ✕) — **тонкой обводкой** + полупрозрачный фон, моно, верхний регистр,
радиус 2–3px.

### 5.10 Строка «командного радио»
Левый бордюр 2px (красный / янтарь-warn), фон `--panel-2`, моно-префикс `P5 ▸`.

### 5.11 Карточка команды / R&D / KPI-чип / строка зачёта
- **Команда:** левый стрип 3px цвета команды, тир (моно), имя, оба пилота (точки
  P5/P6), бюджет/R&D (моно), цель; выбранная — рамка `--red` + красная линия снизу.
- **R&D:** точка ветки, название, описание, текущий бонус (моно, цветом ветки),
  кнопка «Купить · N RP» + тонкий прогресс-бар.
- **KPI-чип:** моно-ярлык сверху (9.5px), число снизу (16px моно), цвет по смыслу.
- **Зачёт:** как строка тайминга, `POS · имя · очки`; твои пилоты — тинт + стрип.

## 7. Экраны

### 7.1 Главное меню
Графитовый фон, **тонкая красная вертикаль** слева как единственный флориш.
Моно-кикер «F1 MANAGEMENT · CO-OP», **сдержанный** титул `APEX DUO` (44–46px, DUO
красным), тэглайн, компактные моно-чипы ролей P5/P6. Колонка кнопок (≈380px) с
левым стрипом 2px: «Продолжить сезон» (зелёный, при сейве) → «Сезон — новый
чемпионат» (красный) → быстрые гонки → хост → `IP + Подключиться`. Версия-чип внизу.

### 7.2 Настройка сезона
H1 «НОВЫЙ СЕЗОН». Три карточки тиров (McLaren / Williams / Cadillac) с узким
цветным стрипом, бюджетом, R&D, целью, обоими пилотами. Сегменты «Сложность» и
«Режим» — выбор подчёркивается красной линией снизу. CTA «Начать сезон →».

### 7.3 Паддок
**Плоская шапка** «APEX DUO · {Команда} · этап N/5 — трасса» + режим-пилюля. Ряд
KPI-чипов (бюджет, R&D, очки конструкторов, след. трасса). Строка морали обоих
пилотов. Две колонки: слева **Чемпионат пилотов** (твои — тинт+стрип), справа
**R&D** (3 ветки). Нижний бар: «Старт гонки → (режим)», «Выйти».

### 7.4 Гоночный HUD — центральный экран
Каркас: **плоская шапка** → **3 колонки** → **нижний бар**.

- **Шапка:** бренд APEX DUO, трасса+тип, скорость ×N; справа — лап-счётчик (моно)
  и статус-пилюля.
- **Левая колонка — Timing Tower (≈300px):** плотные строки 25px с зеброй; твои
  пилоты помечены; лидер с короной; чипы-обводки PIT/OT/CLIP; шина-таблетка 16px.
- **Центр — Карта + Радио:** живая мини-карта (§7.5) сверху; снизу «командное
  радио» (запрос в боксы, предупреждение пит-стэкинга).
- **Правая колонка — КООП-управление (≈312px):** сверху **Командная тактика**
  (живой разрыв P5–P6, общий темп, приказ-свап); ниже две **зеркальные кокпит-
  карты** P5 и P6 — у каждого своя батарея, ERS и «Обгон».
- **Нижний бар:** Пауза · ×1/×2/×5 · «В паддок →» · «Новая гонка».

Кооп-режимы меняют правую колонку: **соло** — панель P5 (+ тактика); **лок.
кооп/хост** — обе; **клиент** — только своя панель P6 + баннер «Вы ведёте P6».

### 7.5 Мини-карта трассы
Процедурный замкнутый контур (свой силуэт на трассу). Полотно: база `#23272E`
(w15) → поверхность `#3D444E` (w9) → гоночная линия `#5A6470` (w2). Поребрики
красный/белый каждый ~7-й сегмент; старт/финиш — «шашка»; стрелка на ~12% круга.
**Маркеры (форма + цвет), уменьшенные:** P5 = диск цвета команды + белое тонкое
гало + золотое кольцо; P6 = **кольцо** (диск с дыркой) + голубое гало; лидер =
тонкая корона + пип; соперник = диск с контуром; клиппинг = сталь; бокс = полое
кольцо; сход = серый ✕; обгон = янтарная аура + длинный трейл. **SC:** приглушённая
жёлтая лента + чип. Полная реализация — в `RACE_VISUAL_SPEC.md`.

## 8. Кооп-визуальный язык (сквозной)

- **P5 = золото `#FFD166`, P6 = голубой `#66C2FF`** — всегда и только так,
  применяются **тихо** (узкий стрип + тинт ≈7%, не яркая заливка).
- Накладываются **поверх** цвета команды (§2.4).
- В таблице/зачёте — слабая подсветка строки + ident-стрип + имя цветом роли.
- На карте — гало-кольцо + форма (диск vs кольцо).
- В панелях — тонкий левый бордюр + шапка роли; панели **зеркальны**.
- **Клиент** видит баннер «Вы ведёте P6» и только свою панель.

## 9. Состояния и обратная связь

| Событие | Визуал |
|---|---|
| Сейфти-кар | приглушённая жёлтая лента на карте + чип `SC` + статус-пилюля |
| Клиппинг | батарея → сталь `--clip` + молния-«пусто»; маркер обесцвечен; чип `CLIP` |
| Запрос в боксы | синий чип `PIT` (обводка) + строка радио |
| Пит-стэкинг | строка радио `warn` (янтарь): «экипаж один, +7 c второму» |
| Быстрейший круг | строка/отрыв фиолетовым + чип `FASTEST` |
| Обгон активен | янтарный чип `OT`, аура на карте, кнопка-обводка залита янтарём |
| Свап-приказ | шевроны между строками P5/P6, лёгкая подсветка |
| Сход (DNF) | строка/маркер гаснут, серый ✕ |
| Финиш | статус-пилюля «ФИНИШ», итог команды в радио, CTA «В паддок →» |

## 10. Движение и анимация

Сдержанно: 120–220ms, ease-out, **без скос-свайпов**.

- Шапка/панели появляются мягким фейдом, не «телевизионным» свайпом.
- **Реордер строк** тайминга — плавный перескок (120–180ms), без мерцания.
- **Один** короткий пульс: фиолетовый на быстрейшем круге, янтарь на включении
  Обгона. Не зацикливать.
- **Заполнение** батареи/износа — анимированное, не скачком.
- `SC` мигает мягко (~1Гц). Движение — акцент на изменении, а не фон.

## 11. Доступность

- **Colour-blind:** каждое отличие дублируется формой/текстом — шина (буква),
  напарник (диск/кольцо), состояние (иконка/чип-текст), SC (чип).
- **Контраст:** при приглушённой палитре особенно следить за AA (≥4.5:1). Не
  ставить `--dim` на `--panel-2` для важного текста — проверить моно-ярлыки 9.5px.
- **Минимальные размеры:** шрифт в HUD ≥ 12px, моно-ярлыки ≥ 9.5px (по AA — на
  грани, проверить), цель клика ≥ 30px.
- Перед хэндофом прогнать через скилл `design:accessibility-review` (WCAG 2.1 AA).
