(ChatGPT, Qwen), идеи Карима про скилы в стиле Football Manager и разбора
игр-аналогов (`COMPETITOR_ANALYSIS.md`), отфильтровано под текущую архитектуру и
отсортировано по приоритету.

- **Дата:** 2026-06-09
- **Статус:** идеи, не обязательства. Ничего из этого ещё не закоммичено.
- **Важно:** этот документ — только запись. Код игры не меняется. Позже отсюда
  можно перенести готовые формулировки в `Apex_Duo_GDD.docx`.
- **Источники:** `ChatGPT` = первый разбор, `Qwen` = второй разбор,

## Легенда

- **Приоритет:** `P1` (делать первым) · `P2` (следующая волна) · `P3` (позже / длинный хвост).
- **Сложность:** `S` < `M` < `L` < `XL`.
- **Флаги:**
  - `[EXT]` — расширяет уже существующую систему.
  - `[NEW]` — новая система.
  - `[DET]` — обязателен детерминизм (см. принципы ниже).
  - `[DECISION]` — требует решения о направлении игры, не просто реализации.

## Сквозные принципы (читать до реализации любой фичи)

1. **Детерминизм — несущая стена.** Любой «риск / случайность / неопределённость»
   идёт **только** через сид-RNG: основной поток `rng` (потики) и отдельный
   `erng` (события, сид через `mix32`). Никакого реального рандома, времени,
   неупорядоченного обхода словарей в сим-логике — иначе ломается host-authoritative
   нетворкинг и Python-харнесс баланса. Этого пункта внешние разборы не учли — он
   не опционален.
2. **Информация: своё — точно, чужое и прогнозы — с погрешностью.** Это снимает
   противоречие между «информационной неопределённостью» (хотим тумана) и
   «читабельностью стратегии» (хотим ясный HUD). Свою машину игрок видит точно;
   деградацию/настрой соперников и прогнозы погоды/износа — с шумом. Точность
   прогноза можно привязать к навыку «обратная связь» пилота и к скаутингу.
3. **Трение в коопе — только через прототип.** Искусственные ограничители
   (attention points, задержки связи) легко делают игру раздражающей, а не
   глубокой. Сначала прототип, потом в дизайн.
4. **Кооп остаётся двухмашинным.** Сейчас модель — «два игрока = две машины +
   общие тактики». Схема «один крутит темп/ERS, другой — питы у одной машины» из
   разбора ChatGPT — это редизайн, а не расширение. Углубляем через **расхождение
   KPI + асимметрию информации**, не ломая двухмашинную основу (см. разбор A ниже).

## Разбор A. Кооп: модель ролей (важное дизайн-решение)

Самый сильный пункт обоих разборов — конфликт ролей. Но есть развилка, которую
внешние разборы не заметили:

- **Сейчас:** два игрока = две машины + общие тактики (деление по пилотам).
- **ChatGPT предлагает:** Инженер крутит темп/ERS, Стратег — питы у **одной** машины
  (деление по функциям). Это **редизайн** кооп-модели.

**Рекомендация:** не ломать двухмашинную основу. Тот же эффект «договариваемся под
давлением» даёт **расхождение KPI + асимметрия информации**:

- Директор (P5): цели команды — очки в Кубке конструкторов, бюджет, отношения со
  спонсорами, общая стратегия.
- Инженер (P6): цели по своей машине — беречь компоненты (RL-1), идеальное окно
  пита, темп/ERS своего пилота, его мораль.
- **Трение возникает само:** общий пит-экипаж (штраф за стэкинг уже есть), общий
  бюджет R&D (MT-2), тим-ордера (CO-3), асимметричная телеметрия (CO-2). Никто не
  видит всю картину — приходится говорить.

Это закрывает идеи ChatGPT 3.2 и 3.4 разом и без переписывания нетворкинга.

## Разбор B. Система скилов в стиле Football Manager (идея Карима)

Хорошая идея и **высокий рычаг**: один фундамент питает личности (DR-3), архетипы
(DR-4), скаутинг/Академию (MT-1), неопределённость (DR-5), развитие и мораль
(уже есть). Ложится на конвенцию проекта «данные — это `const`-словари».

**Шкала.** Хранить 0..1 (как навыки/характеристики трасс в симе), показывать 1–20
(привычно по FM). Маппинг каждого атрибута — на уже существующий член сим-модели.

**Видимые атрибуты (примерные группы):**

- *Скорость:* квалификационный темп → грид; гоночный темп → базовый чистый круг;
  торможение; прохождение поворотов.
- *Боевые:* обгон → «pass credit» в `_resolve_combat`; защита → сопротивление
  обгону; агрессия → риск контакта (связка RL-2); старт → разброс на старте;
  езда в трафике → меньше потерь в грязном воздухе.
- *Менеджмент:* управление шинами → множитель износа; управление ERS → меньше
  клиппинга, эффективнее харвест/деплой; экономия топлива; стабильность → шум
  темпа и шанс ошибки.
- *Условия:* дождь → темп на мокрой (связка TR-1); адаптивность → быстрее выходит
  на пик на новой трассе/сетапе.
- *Технические:* качество обратной связи → точнее направление R&D и **меньше
  погрешность телеметрии** (прямая связка с принципом 2); механическая чуткость →
  меньше износ компонентов (связка RL-1).
- *Ментал/физика:* самообладание → меньше ошибок на рестартах SC и в конце гонки;
  концентрация; выносливость → меньше падение темпа на длинной дистанции; реакция.

**Скрытые атрибуты (раскрываются скаутингом/временем — связка MT-1, DR-5):**

- Потенциал (PA) против текущего уровня (CA) → потолок развития.
- Профессионализм → скорость развития + стабильность морали.
- Амбиции → требует статус №1, давит на контракт (связка MT-4).
- Эго → шанс проигнорировать тим-ордер (связка CO-3, DR-3).
- Характер/темперамент → PR-инциденты, реакция на критику (связка CO-7).
- Воля (determination) → камбэки. «Биг-рейс» темперамент → лучше/хуже в решающих
  гонках. Скрытая волатильность. Травмоопасность → шанс пропуска (связка CO-4).

**Детерминизм.** Атрибуты — статические данные в `f1_2026.gd`. Раскрытие скрытых —
детерминированная функция от сида скаутинга, без реалтайм-рандома.

**MVP, чтобы не делать сразу XL.** Начать с 6–8 атрибутов, которые уже неявно есть
в симе (темп, управление шинами, управление ERS, обгон, защита, стабильность) +
2–3 скрытых (PA/CA, профессионализм, эго). Дальше расширять группами.

## Рекомендованный порядок

1. **P1 — Риск/надёжность:** RL-1 (DNF) + RL-2 (риск-ось) + RL-3 (инциденты ИИ).
   На роадмапе, детерминизм-safe, быстрый выигрыш, настоящие дилеммы.
2. **P1/P2 — Кооп-асимметрия (USP):** CO-2 (телеметрия) + CO-1 (KPI) + CO-5 (пинги)
   + CO-3 (тим-ордера) + **FM-1** (доска ответственностей — формализует разрез
   ролей). Сначала зафиксировать модель из разбора A.
3. **P2 — FM-скилы:** DR-1 + DR-2 как фундамент (большой, но питает половину
   бэклога: DR-3/4/5, MT-1, MT-4, CO-3).
4. **P2 — Читабельность и энергетика:** UX-1, затем EN-2 / EN-3 / EN-4.
5. **P3 — Длинный хвост:** мета для Игрока 2 (MT-1/2/3), погода (TR-1/2),
   политика (MT-4), эндгейм (EG-1/2).
