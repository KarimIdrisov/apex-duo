## 3. Трасса: реальная езда, повороты, скорость (твой пункт 2)  — подумать/спека

Сейчас болиды едут по `lap_fraction` с равномерной визуальной скоростью по контуру
(`track_map.gd`). Чтобы «реально ездили, учитывали повороты и можно было смотреть
скорость в момент времени» — нужен **профиль скорости по трассе**:

- [ ] Дать трассе геометрию с **кривизной по сегментам** (медленно в поворотах,
  быстро на прямых) — из контура `TRACK_SHAPES`/процедурного или отдельным профилем.
- [ ] Визуально: скорость продвижения `d(frac)/dt` зависит от локальной кривизны
  (на прямой быстрее, в повороте медленнее) — болид «тормозит/разгоняется».
- [ ] **Мгновенная скорость (км/ч)** на машину из локального сегмента + темпа; показать
  по наведению/выбору или в телеметрии (точка во времени → скорость).
- [ ] Решить уровень: **косметика** (только визуал скорости) vs **симуляция** (реальная
  позиционная модель, скорость влияет на обгоны/тормозные зоны). Для «смотреть скорость»
  осмысленнее симуляционный вариант → отдельная дизайн-спека перед кодом.
- [ ] Бонус: тормозные зоны как точки обгона (привязать к `_resolve_combat`).
