# Дизайн: красивый 3D-вид гонки «бродкаст» (Apex Duo)

Статус: **спека, к ревью.** Дата: 2026-06-10.
Развивает ресёрч-бриф `docs/3D_VISUALIZATION_RESEARCH.md` (фазы 0–3 уже в проде)
и доводит вид до «красиво» (фазы 4–7). Ничего не реализуется до утверждения спеки.

## 1. Контекст и цель

Сейчас 3D-вид (`race_view_3d.gd` + `track_builder_3d.gd`) — рабочий каркас уровня
«фаза 3»: кубики-болиды кинематически едут по плоской CSG-ленте трассы, одна
статичная изо-камера, базовое окружение. Геометрия уже **движется правильно**
(детерминированный мост `frac → точка на Curve3D`), не хватает только красоты.

Цель — **телевизионная бродкаст-картинка**: это менеджер, игрок смотрит гонку как
по ТВ (он гоночный инженер/директор, не пилот). Значит, читаемость всех 22 болидов
и особенно своих P5/P6 важнее «визцеральности». «Дорого выглядит — дёшево по арту».

**Фундамент уже есть:** 25 точных контуров реальных трасс в `track_shapes.gd`
(порт из f1-circuits-svg, CC BY 4.0) — общий источник для 2D-миникарты и 3D.

## 2. Зафиксированные решения

| Развилка | Решение |
|---|---|
| Уровень амбиции | **Бродкаст-стилизация** (цель B): low-poly + PBR, не реализм |
| Камера | **Гибрид**: ТВ-режиссёр по умолчанию + chase-камера (переключатель и кооп) |
| Арт/модели | **Готовые CC0-ассеты (Kenney)** для машин и декора, **+ фолбэк на примитивы** |

## 3. Принципы и инварианты (неизменны)

- **Чистый вид поверх сима.** 3D только *читает* снапшот, никогда не пишет в сим.
  Детерминизм и хост-авторитет не затрагиваются (правило CLAUDE.md «Sim stays UI-free»).
  Время кадра / порядок обхода узлов 3D не попадают обратно в сим.
- **Контракт `ensure_built` / `set_cars`** сохраняется — интеграция в `main.gd`
  остаётся минимальной, 2D-миникарта остаётся как угловой оверлей / «быстрый режим».
- **Грузим ассеты с мягким фолбэком на примитивы** — проект запускается даже без
  скачанного Kenney-пака (в песочнице бинарные ассеты не проверить).
- **Бюджет производительности:** Forward+ для соло; в split-screen (рендер ×2) —
  более дешёвые тени/частицы/LOD по флагу качества.

## 4. Архитектура и файлы (мелкие, проверяемые модули)

Каждый модуль — одна ответственность, тестируется в изоляции.

- `track_builder_3d.gd` — **апгрейд**: плоская CSG-лента → `ArrayMesh` (поребрики,
  гоночная линия, трава/гравий, рельеф, разметка). Чистая геометрия, проверяется численно.
- `race_view_3d.gd` — оркестратор: вьюпорты, мир, окружение, прокидывание снапшота
  в камеру и риги. Худеет — логика уезжает в модули ниже.
- **new** `race_camera_3d.gd` (`class RaceCameraDirector`) — режимы камеры + авто-режиссёр.
  Чистая логика: на вход трансформы машин + флаги событий + dt → позиционирует `Camera3D`.
- **new** `car_rig_3d.gd` (`class CarRig3D`) — вся визуалка одной машины (меш+фолбэк,
  колёса/подрул, цвет компаунда, состояния, биллборды). `rig.update(car_dict, xf, dt)`.
- **new** `scenery_3d.gd` (`class Scenery3D`) — MultiMesh-декор вдоль кривой по сиду.
- `assets/` — Kenney glTF (машина, декор) + материалы. Загрузчик с фолбэком.

## 5. Контракт данных (что вид читает из снапшота)

Ровно то, что `main.gd._update_track_map` уже кладёт в массив для `set_cars`:
`id`, `frac`, `team_color`, `state` (`run`/`attack`/`clip`/`pit`/`out`), `team`,
`lead`, `slot`, `pos`, `pit_phase`.

**Единственное добавление:** поле **`compound`** (строка soft/medium/hard/inter/wet)
в `_update_track_map` — для цвета дисков, видно стратегию. Сим его знает
(`Driver.compound`, строка из `COMPOUNDS`);
это +1 строка в сборке снапшота, **сим-логику не трогаем**, поле опционально-деградируемое
(нет компаунда → нейтральный диск). Скорость/подрул берутся из дельты `frac` (есть).

## 6. Компоненты

### 6.1 Камера-режиссёр (`RaceCameraDirector`) — даёт максимум «вау»

- **ТВ (по умолчанию):** кольцо «вышечных» камер, сгенерённых из `Curve3D` (через
  каждые ~N метров, смещены наружу+вверх, смотрят внутрь). Режиссёр выбирает вышку,
  лучше всего держащую «субъекта» (лидеры или ваша машина), плавно ведёт её
  (`look_at`-slerp) и передаёт следующей вышке при уходе субъекта. Лёгкое «дыхание»
  FOV + микро-покачивание → живой эфир.
- **Chase:** трейлит цель (P5/P6 или лидер) — точка сзади-сверху по касательной
  кривой, `look_at` на машину. Режим переключателя и для каждого экрана коопа.
- **Облёт/кино:** медленный орбит — пред-старт, пит, повторы, подиум.
- **Авто-режиссёр:** читает события, которые сим уже отдаёт (борьба/обгон у вашей
  машины, пит, сход/Turn-1 incident, выезд SC), подрезает на лучший ракурс, держит
  таймер кадра, возвращается. Приоритет: ваша борьба → SC → ваш пит → борьба за
  лидерство → лидер.
- **Управление:** кнопки на виде (рус.): «Реж.», «P5», «P6», «Лидер», «Облёт».
- Инвариант: камера читает только трансформы/события, ничего не пишет в сим.

### 6.2 Живая трасса (`track_builder_3d.gd`)

Идём по запечённой кривой с шагом, на каждом сэмпле — центр + бок (`basis.x`),
строим `ArrayMesh`:
- тёмный PBR-асфальт + затёртая **гоночная линия** (UV под текстуру);
- **поребрики** красно-белые только в поворотах (логика углов уже есть в
  `track_map.gd` `_draw_kerbs`/`_draw_zones` — портируем);
- **обочина**: трава + бежевый гравий на выходах поворотов (широкий нижний полигон);
- **рельеф**: Y на точках кривой (сначала лёгкая общая волна, потом авторские
  профили вроде Спа/Сузуки/Шпильберга/Остина — у нас теперь есть их реальные формы);
- **разметка**: старт/финиш + решётка, зоны активной аэро/Overtake, тормозные щиты;
- **пит-лейн**: параллельная полоса у старт/финиша (`_pit_path` из 2D уже считает);
- (полировка) **бэнкинг** в быстрых поворотах через поворот сечения.

Возвращает `{curve, road_mesh, kerbs, surrounds, markings, length}`.
Проверяется численно (длина, равномерность шага, замыкание, поребрики-в-поворотах).

### 6.3 Болиды (`car_rig_3d.gd`)

- **Меш:** Kenney open-wheel из `res://assets/cars/` → **фолбэк** на примитивный
  силуэт F1 (из боксов/цилиндров по пропорциям `BODY`/`WHEELS` из `track_map.gd`),
  если ассета нет — вид всегда запускается.
- **Ливрея:** цвет команды на корпусе (per-instance материал) + акцент.
- **Колёса** крутятся ∝ скорости, передние подруливают по касательной (дёшево, живо).
- **Компаунд** на дисках/боковинах (soft красный / medium жёлтый / hard белый /
  inter зелёный / wet синий) — видно стратегию.
- **Состояния** 1-в-1 из 2D: атака → оранжевое свечение/эмиссия; клиппинг → тускнеет;
  пит → в полосе; сход → серый + дымок (`GPUParticles3D`), сдвиг в сторону.
- **Биллборды** (всегда к камере): кольцо P5 (золото)/P6 (сталь), шеврон лидера,
  номер позиции, опц. фамилия.
- **Сглаживание тиков 0.25c:** lerp позиции + slerp ориентации (как в каркасе).

### 6.4 Атмосфера (`race_view_3d.gd` env + `scenery_3d.gd`)

- Тёплый свет, мягкие тени, filmic-тонмаппинг, лёгкий glow/SSAO, DOF только на кино-кадрах.
- **Частицы** (`GPUParticles3D`, дёшево, с лимитом): дым на блокировках/сходе, искры
  на поребриках, пыль вне трассы; **дождь/брызги по влажности сима — если/когда
  погода появится** (сейчас «следующий шаг»; за флагом, не блокирует).
- **MultiMesh-декор:** трибуны, отбойники, стопки шин, деревья, рекламные щиты,
  маршалы — детерминированно вдоль кривой по сиду, ~один draw call на тип, LOD вдали.
- **Сейфти-кар:** обочина/маркеры желтеют, меш SC ведёт «паровозик» (состояние SC в снапшоте).

### 6.5 Кооп split-screen (`enable_split()`)

Два `SubViewport` на одном `World3D` (каркас уже так делает), у каждого свой
`RaceCameraDirector` в **chase по своей машине** (слева P5, справа P6), 2D-оверлеи
сверху. В split-режиме режем тени/частицы/LOD по флагу качества.

### 6.6 Интеграция (`main.gd`)

Переключатель «Вид: 2D ▸ 3D» уже есть. Делаем 3D основным, 2D-миникарту — угловым
оверлеем; в коопе зовём `enable_split()`; добавляем `compound` в снапшот (§5).
`main.gd` — хотспот (см. `docs/WORKFLOW.md`): правки аддитивные/обратимые,
координировать (параллельные таски).

## 7. Фазы (каждая ценна и проверяема сама по себе)

1. **Камера-режиссёр** — `race_camera_3d.gd`. Максимум «вау» сразу, геометрия уже едет.
   **← первый имплемент-инкремент (отдельный план).**
2. **Живая трасса** — `track_builder_3d.gd`: ArrayMesh + поребрики/трава/разметка/рельеф.
3. **Риги машин** — `car_rig_3d.gd`: Kenney + фолбэк, колёса/подрул, компаунд, состояния, биллборды.
4. **Атмосфера** — env + частицы + `scenery_3d.gd` + SC.
5. **Кооп-сплит** — chase по машинам + бюджеты качества.
6. **Полировка** — бэнкинг, кино-DOF, погода (если появится), авторский рельеф, повторы.

## 8. Верификация (под gotcha'и проекта)

- Свежий-файл `gdparse`/`gdlint` на каждый новый/правленый `.gd`
  (в этом окружении: `python -m gdtoolkit.parser|linter`, без лага монтирования).
- `TrackBuilder3D` — численные проверки геометрии.
- `RaceCameraDirector` — как чистая функция (вход: трансформы+события → проверяем цель камеры).
- Рантайм-санити через godot MCP `execute_gdscript` (реальный Godot 4.6.3): сборка
  вида, шаг сима, валидность трансформов, отсутствие ошибок.
- **Визуал — пользователь в редакторе** (движок в песочнице не стартует).
- Гард детерминизма: убедиться, что 3D-слой не пишет в сим.

## 9. Риски и открытые вопросы

- **Fit Kenney под F1:** их CC0-машины аркадно-ралийные; возможно ближайший
  open-wheel + тинт/масштаб, либо качественный примитив-фолбэк. Проверить конкретный меш.
- **Скачивание ассетов:** бинарные паки в песочнице не проверить — в плане явный шаг
  «положить Kenney-пак в `assets/`» с CC0-ссылками + фолбэк, чтобы ничто не блокировало.
- **Перф split-screen** (рендер ×2) — флаг качества.
- **`main.gd` — общий хотспот:** правки аддитивные, координировать с параллельными тасками.
- **Погода** ещё не в симе — рендер дождя за флагом, не блокирующий.
