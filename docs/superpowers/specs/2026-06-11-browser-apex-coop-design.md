# Apex Web — браузерный кооп-F1 (v1 design)

**Дата:** 2026-06-11
**Статус:** утверждён в брейншторме, ожидает финального ревью спека
**Связь с основным проектом:** отдельный эксперимент. Берёт *данные* и *модель сима*
из Godot-прототипа `ApexDuo_Prototype/` (порт, не зависимость). Godot-проект не трогаем.

## 1. Что это

Браузерная кооп-игра в духе Sid Meier по мотивам F1. Два игрока — пилоты **одной
команды** (две машины одной реальной команды 2026), проходят **полный уикенд одной
трассы**: Практика → Сетап → Квала → Гонка. Гонка — реалтайм с синхронной паузой.
Остальные 20 машин грида — AI. Цель — командные очки (сумма финишей двух машин),
но каждый «инженерит» свою машину сам.

Приоритеты v1 (в порядке важности): **играбельность → удобство → простота →
баланс**. Никаких новых систем сверх перечисленного; глубину добавляем позже.

## 2. Решения брейншторма (зафиксированы)

| Вопрос | Решение |
|---|---|
| Кооп-модель | Одна команда, две машины; каждый ведёт свою |
| Ритм гонки | Реалтайм с синхронной паузой |
| Связь игроков | Онлайн, два браузера |
| Деплой / транспорт | **GitHub Pages (чистая статика) + WebRTC P2P** через PeerJS-брокер; браузер-хост авторитетен |
| Фазы уикенда | Практика → Сетап → Квала → Гонка |
| Грид | Реальный 2026, полный (22), порт данных из `f1_2026.gd` |
| HUD гонки | Вариант **B «Фокус на пилоте»**: своя машина крупно, большие кнопки рычагов, гэпы спереди/сзади, таблица 22 — по кнопке |
| Сетап-экран | 3 ползунка + индикатор близости в % + текстовый фидбэк пилота + видимая зелёная риска-идеал + ready-gate |
| Трасса v1 | **Барселона** (technical) — по умолчанию, одна строка данных, меняется тривиально |

## 3. Стек и принципы

- **Чистая статика**: `index.html` + ES-модули + Canvas 2D. Без фреймворка, без шага
  сборки, без npm-зависимостей в рантайме клиента. Открыл файл → работает; `git push`
  → GitHub Pages раздаёт.
- **PeerJS** подключается тегом `<script>` с CDN (единственная внешняя зависимость,
  только для онлайна). Сигналинг — публичный брокер PeerJS (не наш сервер).
- **Хост-авторитетная модель**: только хост гоняет `sim`. Клиент шлёт команды,
  рендерит снапшоты. Совпадает с моделью нетконтента Godot-прототипа.
- **Детерминизм сима обязателен** (как в Godot): seeded RNG, никакого реального
  времени/неупорядоченных итераций внутри сима. Это страхует от рассинхрона и даёт
  воспроизводимость для балансного харнесса.
- **Язык интерфейса — русский**; код/комментарии — английский.

## 4. Структура модулей

Каждый модуль — одна ответственность, общается через явный интерфейс, тестируется
отдельно.

```
index.html      каркас, <canvas>, подключение PeerJS, точка входа main.js
data.js         чистые данные: грид 2026 (порт) + 1 трасса. Никакой логики.
sim.js          чистый детерминированный сим: class Race. Без DOM, без сети.
weekend.js      машина состояний уикенда: PRACTICE→SETUP→QUALI→RACE + ready-gate.
net.js          интерфейс Net + реализации LocalNet (BroadcastChannel) и P2PNet (PeerJS).
ui/lobby.js     создать/войти в комнату по коду; выбор команды.
ui/practice.js  прогоны + фидбэк + индикатор близости.
ui/setup.js     3 ползунка к скрытому идеалу.
ui/quali.js     один быстрый круг с выбором риска; таблица грида.
ui/race.js      HUD гонки (вариант B); рендер из снапшота.
main.js         склейка: Net + weekend + ui; цикл хоста (requestAnimationFrame).
rng.js          seeded LCG + mix32 (порт из Godot, для детерминизма).
```

`sim.js`, `data.js`, `rng.js` — чистые, не знают про DOM/сеть → их же гоняет
балансный харнесс в Node без браузера.

## 5. Модель данных (порт из Godot)

Из `f1_2026.gd` экспортируем **плоский JSON** (результат `team_car()` уже
скомпонован — JS не повторяет логику engine+chassis):

```js
// data.js
export const TEAMS = [
  { name:"McLaren", color:"#ff8000",
    car:{ power:0.97, aero:0.95, energy:0.92, rel:0.97 },   // готовые скаляры из team_car()
    drivers:[ {name:"Норрис", abbrev:"NOR", skill:0.950},
              {name:"Пиастри", abbrev:"PIA", skill:0.942} ] },
  // … 11 команд, 22 пилота (значения skill — из TEAMS в f1_2026.gd)
];

export const TRACK = {            // Барселона
  name:"Барселона", laps:66, lt:80.0, pit:21.5,
  df:0.82, pw:0.55, ot:0.30, abr:1.25, harv:0.58, dep:0.55, sc:0.25, el:0.82
};

export const COMPOUNDS = {        // подмножество без wet/inter в v1
  soft:   { pace:-0.55, wear:2.6, cliff:65 },
  medium: { pace: 0.00, wear:1.7, cliff:78 },
  hard:   { pace: 0.55, wear:1.1, cliff:90 },
};
```

В v1 **режем** из данных трассы: `wet`, `az` (азимут карты не нужен — карту рисуем
проще), `harv/dep` оставляем как модификаторы ERS. `el` (электро-нагрузка) оставляем.

## 6. Модель сима (упрощённый порт `race_sim.gd`)

Держим ровно ингредиенты, делающие 3 рычага осмысленными. Числа — стартовая точка
из Godot-констант (`SKILL_K=3.0`, `CAR_K=1.2`, `DNF_BASE=0.005`, компаунды/режимы
выше), финальная калибровка — балансным харнессом (раздел 10).

**Тик:** фиксированный шаг `STEP=0.25s`, три фазы за тик (как в Godot):

1. **Ход машины** — чистое время круга:
   ```
   lap_time = lt
            − SKILL_K * (skill − 0.5)                       // навык пилота
            − CAR_K  * car_pace_bias(car, track)            // машина×характер трассы
            + compound.pace + tyre_wear_term(wear, compound) // резина + обрыв
            + PACE_MODES[pace].pace                          // темп
            + ERS_MODES[ers].pace + clip_penalty(soc)        // ERS + клиппинг при пустом сторе
            + noise(rng)                                     // малый шум
   ```
   - **Износ** копится: `wear += compound.wear * PACE_MODES[pace].wear / laps_scale`.
     За `cliff`-порогом — резкая добавка к времени (заставляет питоваться).
   - **DNF**: на завершении круга roll `DNF_BASE*(1−rel)` (×1.6 на push). Маленький.
2. **Бой за позицию** (`_resolve_combat`, упрощённо): догнал лидирующую впереди на
   `<COMBAT_GAP=0.8с` → «грязный воздух» (штраф к темпу), копит pass-credit от
   разницы темпа (+буст при ERS=attack); обходит, когда credit превышает
   сопротивление трассы `~ (1 − track.ot)`. **Инвариант (как в Godot): бой пишет
   только относительную позицию на круге, не присваивает номер круга.**
3. **Завершение круга + питы**: списание заряда/износа, исполнение пит-стопа
   (`track.pit` секунд), бухгалтерия кругов.

**Единый стор заряда** (battery v0.6): `soc` 0..100, `ERS_MODES[ers].soc` меняет
за круг (+harvest / −attack), пустой → `clip_penalty`. Никаких deploy_budget.

**Режем из v1** (план на потом): топливо, active-aero детали, сейфти-кар, погода,
сезон/чемпионат, развитие машины (R&D). AI-логика — простая: держит баланс,
питуется по износу за `cliff`, иногда атакует.

## 7. Поток уикенда (`weekend.js`)

Состояния: `LOBBY → PRACTICE → SETUP → QUALI → RACE → RESULT`. Каждый игрок владеет
своей машиной; переход — только когда **обе** машины «готовы» (ready-gate, гейтит
хост).

- **PRACTICE**: до 3 прогонов по 3 круга. Каждый прогон уточняет «индикатор
  близости» (в %) к скрытому идеалу сетапа трассы и даёт текстовый фидбэк пилота
  («не хватает переднего прижима в медленных» и т.п., шаблоны по самой далёкой от
  идеала оси). Зелёная риска-идеал на ползунках появляется после первого прогона.
- **SETUP**: 3 ползунка (Прижим / Передачи / Подвеска), у каждого скрытый идеал
  `0..1` (детерминирован от seed трассы). Близость = среднее по осям → **малый
  бонус к темпу** (до ~0.15 с/кр) + **модификатор износа**. Формула baseline —
  одинаковая для AI и игрока (как в Godot car_setup), чтобы было честно.
- **QUALI**: каждый игрок едет **один быстрый круг**, выбирает уровень риска
  (push сильнее = быстрее, но больше разброс/шанс ошибки-«затупа»). AI-машины
  получают квал-время из той же формулы. Сортировка 22 времён → стартовая решётка.
- **RACE**: реалтайм с синхронной паузой. Рычаги: **Темп** (Save/Norm/Push),
  **ERS** (Harvest/Bal/Attack), **Пит** (когда + компаунд). Командные очки —
  по позициям обеих машин (очки F1: 25-18-15-…).
- **RESULT**: финишный порядок, командные очки, кнопка «ещё уикенд» (тот же сид+1).

## 8. Netcode (host-authoritative, P2P)

- **Комната**: хост создаёт PeerJS-id → показывает код. Клиент вводит код →
  открывается DataChannel. Тонкий интерфейс `Net`: `send(msg)`, `onMessage(cb)`,
  `role` (`host`/`client`).
- **Клиент → хост**: `{type:"command", cmd:"set_pace|set_ers|request_pit|set_setup|
  set_ready|toggle_pause|quali_push", payload}`.
- **Хост → клиент**: `{type:"snapshot", phase, cars:[{pos,gap,tyre,wear,soc,...}],
  paused, …}` ~10–15 Гц. Клиент рендерит снапшот; между снапшотами — лёгкая
  интерполяция позиций на карте.
- **LocalNet** (разработка): `BroadcastChannel` между двумя вкладками одной машины,
  тот же протокол. Переключение реализации — одна строка в `main.js`.
- **Хрупкость P2P принята**: если хост закрыл вкладку — сессия прервалась. Для двоих
  это приемлемо; устойчивый сервер — отдельный путь на потом.

## 9. UI

- **HUD гонки — вариант B**: топбар (круг N/66, позиция, пауза); своя машина крупно
  — компаунд+износ (бар), заряд ERS (бар), гэп спереди/сзади, напарник; большие
  сегментные кнопки Темп/ERS; крупная кнопка «В боксы» (с выбором компаунда);
  таблица 22 — по кнопке/оверлею. Мини-карта — небольшая, точки = машины.
- **Сетап/Практика**: индикатор близости в %, фидбэк пилота, 3 ползунка с видимой
  зелёной риской-идеалом, кнопки прогонов, ready-gate.
- Стиль — тёмный пит-волл, крупные тач-дружелюбные элементы. Рендер HUD — DOM
  (быстро и удобно для кнопок); только мини-карта и точки машин — Canvas.

## 10. Верификация и баланс

- **Балансный харнесс в Node**: `sim.js`/`data.js`/`rng.js` чистые → отдельный
  скрипт `tools/balance.mjs` гоняет N гонок, печатает: разброс времён по тиру
  команд, эффект undercut, размен темп/ERS, частоту обгонов, среднее число DNF,
  чувствительность к сетапу. Калибруем числа здесь **до** UI.
- **Целевые коридоры** (старт от Godot baseline, уточним харнессом): топ-команда
  быстрее аутсайдера на ~1.5–2.5 с/кр; 1-стоп как дефолт на Барселоне, 2-стоп
  жизнеспособен при push; DNF ~1–2 на гонку из 22; обгон возможен, но не
  бесплатен (track.ot=0.30 — обгонять тяжело, как в реале на Барселоне).
- **Детерминизм-тест**: один seed → идентичный результат гонки (порт инварианта
  Godot: завершения круга == номер круга в баре).
- **Ручная проверка онлайна**: две вкладки (LocalNet), потом два браузера (P2P) —
  команды доходят, снапшоты рендерятся, пауза синхронна.
