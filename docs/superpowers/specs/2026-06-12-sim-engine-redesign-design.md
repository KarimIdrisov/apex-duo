# Apex Web — Simulation Engine Redesign (design)

**Дата:** 2026-06-12
**Статус:** утверждён в брейншторме, ожидает финального ревью спека
**Контекст:** углубляем гоночный движок `ApexWeb/`. Убираем батарею/ERS, делаем
гонку настоящим стратегическим состязанием в духе Motorsport Manager / F1 Manager
с FM-моделью команды (атрибуты пилота, показатели болида, персонал). Порт идей из
Godot-прототипа (`race_sim.gd` ATTR_KEYS, `personnel.gd` ROLES, `track_shapes.gd`).

## 1. Цель

Заменить мелкую энергомодель (один SoC-стор) на **топливо как жёсткий ресурс** и
выстроить вокруг трёх связанных решений — **темп (шины), мотор (топливо), стратегия
питов** — глубокий, но читаемый сим. Гонку определяют: управление шинами (прогрев,
износ, обрыв), управление топливом (вес, расход, «доехать»), обгоны (слипстрим/
грязный воздух, посекторно), события (сейфти-кар, старт, погода) и **умный AI**. Над
всем — FM-слой: атрибуты пилота, болид, персонал у игрока И у AI.

Приоритет: **играбельность и вызов** (стратегия должна «кусать» — лидер не
выигрывает в 0 питов), при сохранении детерминизма и хост-авторитетного нетконтента.

## 2. Решения брейншторма (зафиксированы)

| Вопрос | Решение |
|---|---|
| Батарея/ERS | **Убрать** (режимы, SoC, клиппинг, ERS-рычаг в UI) |
| Рычаги в гонке | **Темп** (управление шинами) + **режим мотора** (топливо push/save) |
| Топливо | **Жёсткий ресурс**: запас, расход×режим, вес, пустой бак = сход |
| Шины | **Прогрев + кривая износа + обрыв**; компаунды (вкл. inter/wet) |
| Обгоны | Эмерджентно: **слипстрим + грязный воздух**, посекторно; без кнопки |
| События | **Сейфти-кар + старт-инцидент + погода** — всё в ядре |
| Трасса | **3 сектора + мини-сектора**, straightness из реального контура; посекторный fit и бой; мини-секторное время на дашборд |
| Команда (FM) | **13 атрибутов пилота**, **5 показателей болида**, **персонал** (стратег + пит-крю) у игрока и AI |
| AI | Планирует пит-стратегию, управляет темпом/топливом, реагирует на SC/погоду, атакует/обороняется |
| Подход | **Развить** детерминированное ядро (3 фазы/тик, 2 RNG-потока), вынести чистые модули |

## 3. Архитектура

Сохраняем детерминированный per-tick цикл (`STEP=0.25с`, 3 фазы), два RNG-потока
(`rng` — темп/износ/шум; `erng` от `mix32(seed)` — SC/инцидент/погода/DNF),
хост-авторитет, **инвариант боя** (пишет только `lapFrac`, не `lap`).

Чтобы `sim.js` не распух — выделяем чистые модули с явными интерфейсами:

```
src/sim.js          оркестратор: тик, фазы, order(), снапшот-поля
src/track.js        сектора/мини-сектора: straightness из TRACK_PATH, sample(lapFrac)
src/tyres.js        компаунды, прогрев(temp), кривая износа+обрыв, wet_opt
src/fuel.js         запас, расход×режим, вес→время, «доехать»
src/weather.js      детерминированный таймлайн влажности + slick↔inter↔wet
src/overtake.js     слипстрим/грязный воздух/pass-credit (посекторно)
src/team.js         генерация атрибутов пилота (13), болид (5), персонал (роли)
src/ai_strategy.js  пит-план + темп/топливо/реакции на SC/погоду
data.js             таблицы: грид(overall+черты), трассы(сектора), компаунды, тюнинг
```

Каждый модуль — чистая логика без DOM/сети → гоняется в Node (тесты + балансный
харнесс). `sim.js` импортирует их и проводит влияние **только через термы времени
круга, бой и пит-исполнение** (как `team_car()`/`Personnel` в Godot).

## 4. Модель времени круга (за тик, для одной машины)

```
lap_time = Σ_по_мини-секторам( seg_time )                  // §5 трасса
  − DRIVER_PACE_K · (drv.pace − 0.5)                        // навык-гонка (атрибут pace)
  + tyre_term(compound, wear, temp)                          // §6 шины
  + pace_term(pace_mode, drv)                                // рычаг шин
  + engine_term(engine_mode)                                 // рычаг топлива
  + fuel_weight_term(fuel_kg)                                // §7 вес
  + dirty_air_term − slipstream_term                         // §8 обгон (локально по сектору)
  + weather_term(wetness, compound, drv.wet)                 // §9 погода
  + setup_bonus
  + noise(drv.consistency)                                   // меньше у стабильных
```

`seg_time` (мини-сектор) несёт **посекторный fit болида**: power помогает там, где
`straightness→1` (прямые), aero — где `straightness→0` (повороты). Сумма по кругу
эквивалентна старому `CAR_K`-смещению, но теперь распределена по трассе.

## 5. Трасса — 3 сектора + мини-сектора (`track.js`)

- Трасса: **3 сектора**, каждый из **N мини-секторов** (~5–6, итого ~18). Мини-сектор
  = `{straightness 0..1, len_frac}`. Сумма `len_frac` = 1.
- **Барселона:** straightness выводим из `TRACK_PATH` — кривизна полилинии по бинам
  длины (малый угол между сегментами → прямая; большой → поворот). Границы 3 секторов
  — по равным долям длины (или заданные). Для других трасс — авторские таблицы.
- `sample(lapFrac) → {miniIndex, sector, straightness}` — где машина сейчас (для боя).
- **Мини-секторное время:** для каждой машины — время прохождения каждого мини-сектора
  за круг (доля `seg_time`), копится по ходу круга; на завершении — 3 секторных времени
  + 18 мини. **Подсветка на дашборде:** фиолетовый = лучший в сессии, зелёный = личный
  лучший, жёлтый = медленнее (как на референсе formula1dashboard).

## 6. Шины (`tyres.js`)

- Компаунды (`COMPOUNDS`): **soft / medium / hard** (сухо) + **inter / wet** (дождь).
  Поля: `pace` (soft быстрее), `wear` (скорость износа), `cliff`, `warm` (скорость
  прогрева), `wet_opt` (оптимальная влажность).
- **Прогрев (`temp` 0..1):** свежая шина холодная (после пита / на старте). `temp`
  растёт к 1 каждый круг со скоростью `warm × drv.tyre`. Холодный штраф к времени =
  `(1 − temp) · WARMUP_PEN`. → выезд из боксов медленный → **undercut/overcut**.
- **Износ + обрыв:** `wear += compound.wear · pace_mult · (2 − drv.tyre_norm)` за круг.
  Потеря темпа — плавная кривая до `cliff`, дальше круто (заставляет питоваться).
- Пит: свежая (`wear=0`, `temp=cold`), компаунд по выбору. `drv.tyre` (атрибут) и
  `car.tyre` (доброта шасси) замедляют износ и ускоряют прогрев.

## 7. Топливо (`fuel.js`) — жёсткий ресурс

- Старт: `fuel_kg` = расход_на_круг · laps · (1 + margin_track). Расход за круг =
  `BURN_BASE · engine_mult / car.fuel`. 
- **Вес:** `fuel_weight_term = fuel_kg · WEIGHT_K` (полный бак ≈ +X с/кр в начале,
  легчает к финишу).
- **Режимы мотора** (рычаг): **Save / Standard / Push** — `{pace_offset, burn_mult}`.
  Push быстрее, но жрёт; Save медленнее, бережёт.
- **Жёстко:** нужно доехать с `fuel ≥ 0`. Индикатор «кругов топлива vs осталось». При
  нехватке — игрок/AI вынужден Save (большая потеря темпа `FUEL_STARVE_PEN`); **пустой
  бак → сход (DNF)**. `drv.smoothness` и `car.fuel` улучшают экономию.

## 8. Обгоны (`overtake.js`) — эмерджентно, посекторно

Модель удержания + pass-credit (как сейчас), но обогащённая, локально по straightness:
- **Слипстрим/тоу:** в `<COMBAT_GAP` на прямом мини-секторе догоняющий получает буст
  темпа (× `straightness` × `car.power`) → закрывает гэп и копит credit в зоне торможения.
- **Грязный воздух:** в `<~1.5с` в поворотном мини-секторе — потеря темпа И **+износ
  шин** (× `(1−straightness)`). Нельзя камповать: либо атакуй, либо отпусти охладить
  резину. Убирает «паровозы».
- **Pass-credit:** копится от (разницы темпа + слипстрим) × `drv.overtaking`; сопротивление
  = базовое `(1−track.ot)` × `defender.defending`. Нормируем посекторно так, чтобы
  среднее по кругу вело себя как `track.ot` → обгоны концентрируются в зонах торможения.
- «Атака» игрока = Push темп + Push мотор (платишь шинами/топливом) рядом с соперником.

## 9. События (`weather.js` + sim)

- **Старт-инцидент:** малый шанс по трассе (`erng`), завязан на `drv.starts`/`aggression`
  — потеря времени, редкий сход у пары машин на 1 круге.
- **Сейфти-кар:** ролл по `track.sc_prob` (`erng`); пелотон собирается (гэпы схлопываются
  к узкому поезду за N кругов), пит дешевле (меньше потеря) → стратегическое окно;
  рестарт через `SC_MIN_LAPS`. Поля состояния SC в снапшоте.
- **Погода:** детерминированный **таймлайн влажности** `wetness(lap)` от сида (сухо →
  возможный дождь → мокро → высыхание). Кроссовер: слики тонут при `wetness>порог`
  (большой штраф через `wet_opt`), нужны inter/wet; трасса сохнет — слики возвращаются.
  `drv.wet` улучшает мокрый темп. Реактивные пит-решения (игрок и AI).

## 10. FM-модель команды (`team.js`, `data.js`)

### Атрибуты пилота (13, шкала 1..20 → нормализуем 0..1)
`pace, quali, tyre, overtaking, defending, consistency, composure, aggression,
discipline, wet, starts, race_iq, smoothness`.

| Атрибут | Эффект |
|---|---|
| pace | базовый темп в гонке |
| **quali** | темп на одном быстром круге (питает `quali.js`, отдельно от pace) |
| tyre | износ ↓ + прогрев ↑ |
| overtaking | накопление pass-credit (атака) |
| defending | сопротивление обгону |
| consistency | шум ↓ (меньше ошибок) |
| composure | держит темп под давлением/в борьбе |
| aggression | атака ↑, риск инцидента ↑ |
| discipline | ошибки/срывы ↓ |
| wet | темп в дождь |
| starts | старт + избегание старт-инцидента |
| race_iq | качество решений AI-пилота; игроку — точнее радио-подсказки |
| smoothness | экономия топлива и резины |

**Генерация:** существующий `skill` остаётся как `overall` (якорь). 13 атрибутов
**генерятся детерминированно** вокруг `overall` (+ сид от abbrev) с парой «фирменных»
черт у звёзд (напр. VER — quali/overtaking, HAM/ALO — wet/race_iq). Никакого ручного
прописывания 22×13.

### Показатели болида (5; energy убран)
`power` (мотор/прямые), `aero` (прижим/повороты), `reliability` (DNF), `tyre`
(доброта шасси к шинам → износ↓), `fuel` (экономичность → расход↓). Композятся из
движка+шасси (как `team_car()`); в гонку идут только эти скаляры.

### Персонал (`team.js`, у игрока И у AI; sim-роли)
- **Стратег** (strategy, composure, adaptability) → острота пит-решений AI + скорость/
  качество реакции на SC и погоду; игроку — качество рекомендованной стратегии (хинты).
- **Пит-крю** (шеф-механик + 4 над-бортовых) → **время пит-стопа** (быстрее/медленнее
  базового `track.pit`), разброс, малый шанс косяка. 5 ролей → 1 сим-скаляр (как M4).
- Генерится детерминированно из «силы команды» + per-role сид → у соперников/AI тоже
  есть штат (симметрично). Меняемый состав (найм/апгрейд) — мета на потом, вне ядра.

## 11. AI-стратегия (`ai_strategy.js`)

Для каждой AI-машины (и как подсказка игроку):
- **Пит-план:** компаунды + целевые круги стопов из дистанции/трассы; подстройка —
  undercut/overcut под ближайших соперников, пит под SC, переобувание под дождь.
  Острота = `team.strategist`.
- **Темп/мотор:** push в чистом воздухе/в атаке, беречь шины в лидерстве/грязном
  воздухе; Save топлива при нехватке. Завязано на `drv.race_iq` + `strategist`.
- **Бой:** атака/оборона через общую эмерджентную модель (§8) с атрибутами пилота.
- Сложность (difficulty) масштабирует остроту решений и шум AI.

## 12. Детерминизм, снапшот, удаляемое

- Детерминизм load-bearing: без реального времени/неупорядоченных итераций в симе.
  `erng` обслуживает SC/инцидент/погоду/DNF; погодный таймлайн — функция сида.
- **Снапшот** (хост→клиент) добавляет: `fuel`, `fuelLaps`, `engineMode`, `tyreTemp`,
  `wetness`, `scActive`/`scLapsLeft`, секторные/мини-секторные времена + их подсветку.
  Существующие поля (pos, +/-, pit, tyre, tyreAge, gap, int, last) сохраняются.
- **Удаляем:** `ERS_MODES`, `soc`, `CLIP_PEN`, ERS-рычаг в HUD → заменяем **режимом
  мотора** (Save/Std/Push). Комбат-бонус «ERS-attack» → заменяется Push-темпом/мотором.

## 13. Тесты и баланс (харнесс)

Чистые модули → headless под Node. Расширяем `tools/balance.mjs` коридорами:
- **Стратегия кусает:** топ-команда НЕ выигрывает в 0 питов; оптимум — 1–2 стопа;
  пит-тайминг ±2 круга меняет исход.
- **Undercut:** ранний пит на горячей резине отыгрывает ~`X` с у того, кто на холодном
  выезде (целевой коридор).
- **Топливо:** push-всю-гонку → не доезжает; разумный save-then-push быстрее «середины».
- **SC:** частота ≈ `sc_prob`; под SC пелотон собирается, пит-окно выгодно.
- **Погода:** круг кроссовера разумен; кто вовремя переобулся — выигрывает.
- **Сектора:** power-болид быстрее в straight-секторах, aero — в поворотных (мини-сектора
  это показывают).
- **FM-влияние:** высокий `tyre`/`smoothness` удлиняет стинты; сильный пит-крю экономит
  время; quali≠pace даёт «квалифаеров vs гонщиков».
- **Детерминизм:** один сид → идентичная гонка; инвариант боя (`lapFrac`-only).

## 14. Фазы реализации (план строим по ним; спек — полный идеал)

1. **Топливо:** убрать батарею; `fuel.js` (запас/расход/вес) + режим мотора + UI-замена.
2. **Шины v2:** `tyres.js` прогрев + кривая + обрыв → undercut.
3. **Трасса/сектора:** `track.js` straightness из контура; посекторный fit; мини-секторное
   время → дашборд (подсветка).
4. **Обгоны v2:** `overtake.js` слипстрим + грязный воздух, посекторно.
5. **События:** старт-инцидент + сейфти-кар.
6. **Погода:** `weather.js` таймлайн + slick↔inter↔wet + высыхание.
7. **FM-команда:** `team.js` 13 атрибутов пилота (генерация), 5 показателей болида,
   персонал стратег+пит-крю; провязать атрибуты в термы.
8. **AI-стратегия:** `ai_strategy.js` пит-план + темп/топливо + реакции.
9. **Ребаланс + харнесс-коридоры + UI** (топливо/мотор/погода/сектора в HUD).

## 15. Вне scope (на потом)

Полноценная мета-команда с наймом/трансферами персонала и пилотов; развитие болида
(R&D) по ходу сезона; чемпионат/деньги; повреждения машины/компонентов сверх DNF;
квалификация как многосегментная сессия (Q1/Q2/Q3) — пока один быстрый круг от `quali`.
