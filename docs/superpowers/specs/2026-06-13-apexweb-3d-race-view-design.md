# Apex Web — 3D-вид гонки, v1 (дизайн-спек)

- **Дата:** 2026-06-13
- **Статус:** approved на брейншторме, готов к плану реализации
- **Область:** `ApexWeb/` (браузерный co-op прототип), race-экран
- **Затрагивает:** новый `src/ui/race3d.js`; правка `src/ui/race.js` (слот канваса + переключатель 2D/3D); `index.html` (импорт Three.js). **Сим / netcode / детерминизм — НЕ трогаем.**

## Контекст

ApexWeb — браузерная co-op версия (static Pages + WebRTC P2P), детерминированный покруговой сим. Текущий race-экран (`src/ui/race.js`) — timing-дашборд: SVG-миникарта реальной трассы (машины-точки + баттл-линии), лидерборд, радио-лента, рычаги, тёмная тема (HeroUI).

Ресёрч движка/визуала Motorsport Manager (этот же сеанс) показал: в MM 3D-гонка — это рендер поверх статистического сима («races contested by coloured dots»), а само 3D — «полиш без геймплейного преимущества». Владелец осознанно выбрал **полное 3D ради красоты («как в MM»)**, приняв, что это чистый визуал и отдельный мини-проект.

## Решения (закрыты на брейншторме)

1. **Направление:** полное 3D (Three.js/WebGL), не 2.5D.
2. **Камера v1:** орбитальный обзор всей трассы (вид сверху-сбоку, всё поле в кадре). Дальше «по очереди» в ТУ ЖЕ сцену: камера погони → трексайд-кинематик.
3. **Лэйаут v1:** drop-in — 3D-канвас встаёт на место SVG-миникарты в правой панели. Hero-лэйаут (3D как главный экран) — следующий шаг.

## Главный принцип архитектуры

3D — **только новый слой рендера**. Сим, детерминизм, snapshot-протокол, netcode — без изменений. 3D-рендер потребляет те же данные, что и SVG-карта.

В `race.js` уже есть конвейер: на каждый снапшот (~12 Гц) в `ctx._buf[idx]` пушится `{prog, t}`; rAF-цикл сглаживает позицию (`sampleBuf`) и кладёт машину на трассу (`pointAt`). Новый рендер **переиспользует эти буферы** — меняется лишь превращение `frac` → картинка (вместо SVG-точки на 2D → болид в 3D).

## Компоненты

### `src/ui/race3d.js` (новый модуль)
- `init(canvas, ctx)` — строит сцену один раз; запускает свой rAF-цикл; каждый кадр читает `ctx._buf` / `ctx._meta` (их по-прежнему наполняет `race.js`).
- `dispose()` — остановить rAF, освободить GL-ресурсы при уходе с экрана.
- Связность: только **чтение** `ctx._buf` / `_meta` / `_pit` / snapshot-флагов (`scActive`). В сим не пишет.

### Меш трассы (из `TRACK_PATH`)
- `TRACK_PATH` (нормализованная осевая 0..1, та же, что у 2D-карты) кладётся на плоскость XZ (y=0), центрируется по центроиду.
- **Лента:** по каждой точке считаем 2D-нормаль (логика как `normalAt` в `race.js`), раздуваем на ±halfWidth → триангулируем ribbon.
- **Декор:** светло-серые кромки, белая стартовая линия, секторные тинты (S1/S2/S3 — те же `SECTOR_COL`), короткий пит-спур. Поребрики/kerbs — опционально v1.
- **Материал:** тёмный матовый асфальт; плоско.

### Болиды
- Один общий стилизованный low-poly меш («клин» с тёмным кокпитом), инстансы, перекраска по команде (`c.color`).
- **Позиция:** `frac` (из `sampleBuf`) → 3D-точка на осевой; ориентация по касательной; приподнят над лентой.
- **Подсветка:** твои 2 машины — кольцо/ярче; лидер — золотой контур; остальные — цвет команды (маппинг как в `race.js`: `isLeader`, `player`/`isPlayer`).
- **Состояния:** `inPit` → парковка на пит-спур (как `PIT_STOP` в 2D); `retired` → скрыть; flash при отыгрыше позиции — опционально.

### Орбитальная камера
- Перспективная; угол сверху-сбоку; дистанция считается из bbox трассы (всё поле в кадре).
- Управление: drag-to-orbit мышью (свой мини-контроллер или `OrbitControls`). Авто-вращение — выкл по умолчанию (укачивает), опционально.

### Свет / фон
- ambient + 1 directional; плоско, без теней (опц. blob под машиной). Фон тёмный под тему (`#0a0a0c`).

### Fallback / переключатель
- SVG-карту **не удаляем**. Переключатель 2D/3D в шапке дашборда. Если WebGL недоступен или флаг 3D off → 2D-карта.

## Технологии

- **Three.js через ESM-CDN** прямым импортом (`import * as THREE from "https://esm.sh/three@<pin>"`) — без сборки, в духе текущего peerjs-через-unpkg. **Запинить конкретную версию.**
- Канвас в DOM-слоте, где был `<svg>`.

## Что переносим из 2D в v1

Подсветка лидера/игрока; парковка в боксах; жёлтый SC-тинт трассы. Баттл-линии — опционально.

## Производительность

1 ribbon + 22 low-poly болида — для WebGL тривиально, 60 fps. Инстансы болида, без теней. `dispose()` при уходе с экрана.

## Проверка

- Сим / детерминизм / netcode не меняются → их рискам ничего не грозит (регрессий по балансу нет by construction).
- **Чистая математика** нового модуля (осевая→3D-точка, построение ленты, ориентация по касательной, расчёт камеры из bbox) — юнит-тесты в духе `tests/track.test.js` (детерминированно, без GL).
- **Визуал headless не проверяется** → финальная приёмка владельцем (F5 / открыть `index.html`). Установленный паттерн «UI — owner playtest».

## Критерии готовности v1

1. Race-экран → правая панель показывает 3D-канвас с трассой-лентой (Барселона), на месте миникарты.
2. 22 болида плавно едут по ленте в 3D из живого сима; твои 2 подсвечены, лидер золотой.
3. Орбитальная камера держит всю трассу в кадре; drag-to-orbit работает.
4. SC-тинт и парковка в боксах работают; ~60 fps; сим/netcode/детерминизм без изменений.
5. WebGL недоступен / 3D off → автоматически 2D-карта, без падений.
6. Новая чистая математика покрыта юнит-тестами; `npm test` зелёный.

## Вне скоупа v1 (следующие шаги «по очереди»)
