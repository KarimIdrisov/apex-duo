# Apex Duo — архитектура симуляции гонки

Точное описание гоночного ядра (`ApexDuo_Prototype/race_sim.gd`, класс `RaceSim`)
на момент engine v0.3. Документ — основа для ревью: в конце перечислены известные
упрощения и «спящий» код, которые стоит обсудить. Идентификаторы и формулы — как в
коде. Если меняешь логику ядра — обнови и этот файл.

---

## 1. Принципы

- **Детерминизм.** Тот же `seed` → та же гонка. Основа хост-авторитетного неткода
  (только хост крутит `RaceSim`) и Python-харнесса баланса. Запрещено вносить
  недетерминизм (реальное время, неупорядоченный обход словарей, влияющий на сим).
- **Ядро без UI.** `race_sim.gd` — чистая логика и числа. `main.gd` только читает
  состояние и шлёт команды. Все тюнинг-числа — `const`-словари вверху файла.
- **Кооп Mode A.** Две машины команды управляются людьми (id 4 = Директор, id 5 =
  Инженер). Игрок задаёт **директивы**, пилот их исполняет (или нарушает по
  характеру) — см. §8–9.
- **Вход симуляции = пилот + болид.** R&D-развитие команды в сим **не** заходит
  (см. §10, §16); по задумке R&D должен качать болид, а болид влияет на гонку.

## 2. Цикл и фазы

`step(dt)` с фиксированным шагом `STEP = 0.25 c` (вызывается из `main._process`,
для быстрой симуляции — крупными шагами). Порядок одного тика:

1. `_update_safety_car()` — состояние сейфти-кара (§12).
2. **Фаза 1 — свободный темп.** Для каждой машины по порядку: ИИ-/игрок-мозг
   выбирает темп и энергорежим (§8–9), считается `current_laptime` (§3), катится
   ролл **схода (DNF)** (§5), машина продвигается `dt/lt`, копится износ шин,
   обновляется батарея (§4), катится ролл мелкой **ошибки** (лок-ап → +0.25 пита).
3. **Фаза 2 — колесо-в-колесо** (`_resolve_combat`, §6), пропускается под SC.
4. **Фаза 3 — завершение кругов**: топливо−1, `best_lap`, питы (§7), решение об
   исполнении директивы на новый круг (`obey`, §9), финиш на последнем круге.

`order()` сортирует: финишировавшие по `finish_time`, остальные по `progress()`
(= `lap + lap_frac`). Сошедшие (DNF) получают `finish_time = 100000 − lap`, поэтому
классифицируются позади всех доехавших, но по числу кругов между собой.

## 3. Чистое время круга — `current_laptime(d)`

Темп в «чистом воздухе»; эффекты борьбы (грязный воздух, Overtake, кто кого
обошёл) — отдельно в `_resolve_combat`. Под сейфти-каром возвращается
`base_laptime × SC_PACE_MULT` (1.40) и остальное не считается. Иначе складываются:

| член | формула | смысл |
|---|---|---|
| база | `track.base_laptime` | эталон круга трассы |
| скилл | `− skill × 1.0` | совмещённый темп пилота (0..1) |
| **характер машины** | `− (car_power − car_aero) × (track.power − track.downforce) × CAR_K` | силовая машина быстрее на силовых трассах, аэро — на прижимных; в среднем ноль (§10) |
| компаунд | `+ COMPOUNDS[c].pace` | soft −0.55 / medium 0 / hard +0.55 |
| темп-режим | `+ PACE_MODES[m].pace` | conserve +0.45 / balanced 0 / push −0.45 |
| энергия | клиппинг `+ CLIP_PENALTY×(0.6+0.8·power)`, иначе `+ ERS_MODES[e].pace` | разряженная батарея медленнее; харвест +0.30 / attack −0.38 |
| тимордер | `+ 0.8` если `yield_laps > 0` | пропустить напарника |
| износ | `+ wear×0.012`, за клиффом `+(wear−cliff)×0.10` | линейный + обрыв |
| топливо | `+ fuel_laps × 0.018` | тяжёлый бак в начале (~0.95 c на 53 круга) |
| шум | `± 0.025×(1.25 − consistency×0.6)` | стабильный пилот — уже разброс |

## 4. Энергомодель 2026 (батарея SoC)

`soc` 0..100, `soc_max` (сейчас всегда 100). Обновление `_update_soc`:
- **Клиппинг:** при `soc ≤ 0` → `clipped = true` (теряет ERS-буст, медленнее),
  принудительный реген `PASSIVE_REGEN×(0.5+harvest)×harvest_mult`; снимается при
  `soc ≥ SOC_RECOVER` (20) — гистерезис, чтобы не дёргалось.
- **Режимы** `ERS_MODES`: harvest (+6 SoC/круг, медленнее), balanced (0),
  attack (−6.5 SoC, быстрее). Реген масштабируется `(0.5+track.harvest)`, расход —
  `(0.6+0.8·track.deploy)`.
- **Overtake-буст** (преемник DRS): работает только в окне `OT_GAP_S = 1.0 c` за
  соперником и при `soc > OT_MIN_SOC` (14); даёт `OT_PACE` в борьбе (§6), льёт
  батарею `OT_DRAIN×(0.6+0.8·deploy)`.
- `harvest_mult` теперь идёт от **энергоэффективности ДВС** (§10), ≈0.94..1.09.

## 5. Надёжность и сходы (DNF)

В фазе 1 (не под SC): `rng.unit() < DNF_BASE × (1 − reliability) × stress × dt/lt`,
где `stress = 1.6` в режиме push, иначе 1.0. `reliability` = ДВС × шасси (§10).
При срабатывании — `finished/dnf = true`, классификация позади доехавших. Цель
баланса: ~1.8 схода/гонку на Монце, ~2.6 на Монако (push повышает риск).

## 6. Колесо-в-колесу — `_resolve_combat(dt)`

Модель **удержания позиции** (заменила свободный обмен прогрессом, при котором шум
гонял машины в бесконечные обгоны). Идём по бегущему порядку спереди назад; для
пары (A впереди, B сзади) в пределах `COMBAT_GAP = 0.8 c`:

1. **boost**: если у B эффективен Overtake — `−OT_PACE×(0.4+0.6·power)×(0.35+0.65·overtaking)`.
2. **edge** = `(A.last_lt − B.last_lt) + boost` — преимущество B по темпу (c/круг).
3. **credit** (накопленное давление): растёт `(edge − PASS_DEADZONE) × atkf × dt`,
   где `atkf = 0.7 + overtaking×0.5 + aggression×0.2`; если B не быстрее — credit
   тает на `CREDIT_DECAY×dt`.
4. **Обгон** при `credit ≥ _pass_resist() + defending(A)×3.5`, где
   `_pass_resist = 4 + 9×(1 − track.overtaking)` (Монако ~12.5, Монца ~5.4). B
   проходит вперёд, `passes_made += 1`. Иначе B **придержан** на `MIN_GAP_S` (0.25 c)
   за A.

Так обгон **заработан** и **трасса-зависим**: на Монако почти нереально, на Монце
легко. Атрибуты `overtaking`/`aggression` помогают атакующему, `defending` — обороне.

## 7. Питы

- **Игрок:** заявка `request_pit(id, compound)` исполняется на следующем круге.
- **ИИ:** реактивно по износу — `wear ≥ ai_pit_wear` (рандом 55..72) при остатке
  >6 кругов и `pit_count == 0`, либо `wear ≥ 92`; компаунд по остатку (hard/med/soft).
  Под сейфти-каром — оппортунистический дешёвый пит (`wear > 30`).
- **Стоимость:** `track.pit_loss`, под SC × `SC_PIT_FACTOR` (0.55). Командный экипаж
  один: второй заезд в том же окне ловит `PIT_STACK_PENALTY` (+7 c) — «разводите
  заезды!». Андеркат работает естественно: свежие шины быстрее.

## 8. Модель пилота

- **`skill`** 0..1 — совмещённый базовый темп (вычитается из круга).
- **12 атрибутов** (1..20, `ATTR_KEYS`), детерминированно из скилла + сид личности
  (`gen_attributes`): полоса ~6..18 + один из 5 архетипов (агрессор / берегущий
  шины / нестабильный быстрый / оборонец / нейтральный). Хуки в симе:
  `consistency`→шум круга и риск; `composure`→риск ошибки; `tyre`→износ и ранний
  conserve; `overtaking`/`aggression`→credit и атака; `defending`→сопротивление
  обгону; `discipline`→исполнение директив.
- **Слой сезона** добавляется в `main._make_sim` поверх болида **только пилотный**:
  `dev_of` (развитие) + `morale_mod` (мораль) к скиллу, `trust = morale` (§9).
- **Спящие атрибуты:** `wet` и `starts` генерируются и показываются в профиле, но в
  симуляции **не используются** (нет дождя и старта с места) — см. §16.

## 9. Гибрид-директивы (кооп-управление)

Игрок (инженер) задаёт **директиву**, пилот её исполняет:
- `dir_pace` (push/balanced/conserve) и `dir_intent` (free/attack/hold) — это
  приказы, не прямое управление. `set_pace`/`set_intent`/`set_team_pace`.
- `_player_brain`: при `obey` пилот едет по директиве (attack → Overtake в окне;
  hold → не атакует, бережёт; free → ситуативно). При неповиновении — по характеру
  (агрессор «педаль в пол», осторожный сбрасывает).
- `_roll_obey` (раз в круг): `p = clamp(0.45 + discipline×0.4 + trust/100×0.3 −
  cost, 0.2, 0.97)`, где `cost = aggression×0.5` за приказ беречь/держать. Нарушение
  даёт радиособытие «едет по-своему».

## 10. Модель болида: двигатель + шасси (`f1_2026.gd`)

Болид = **ДВС** (общий поставщик) ⊕ **шасси** (своё у команды), компонуются в
`team_car()`:
- **`ENGINES`** (ключ = `pu` команды, делят клиенты): `power`, `energy`, `rel`.
  Mercedes → Макларен/Мерседес/Уильямс/Альпин; Ferrari → Феррари/Хаас/Кадиллак;
  Honda → Астон; Red Bull Ford → Ред Булл/Рейсинг Буллз; Audi → Ауди.
- **`CHASSIS`** (на команду): `aero`, `rel`.
- Композиция: `power = ДВС.power`, `aero = шасси.aero`, `energy = ДВС.energy`,
  `rel = ДВС.rel × шасси.rel`. Сим читает `car_power/car_aero/reliability` и
  `harvest_mult` (из `energy`).
- **Характер** = `power − aero` (уклон): силовая связка выигрывает на скоростных
  трассах, аэро — на прижимных (член в §3, `CAR_K = 2.5`). Абсолютная сила — в
  скилле пилота. Свинг по реальной таблице 0.04..0.68 c/круг.
- Таблица **статична**: развитие через R&D пока не подключено (§16).

## 11. Трассы

`Track` несёт 8 характеристик 0..1: `downforce`, `power`, `overtaking`, `harvest`,
`deploy`, `abrasion`, `sc_prob`, `wet_prob` (+ `laps`, `base_laptime`, `pit_loss`).
Два источника: **`REAL_TRACKS`** (10 реальных: Монако, Монца, Спа, Сильверстоун,
Сингапур, Бахрейн, Хунгароринг, Сузука, Баку, Зандворт) и процедурные
**`ARCHETYPES`** (power/street/highspeed/technical/modern) с детерминированным
джиттером. `real_calendar` берёт `n` трасс перетасовкой Фишера–Йетса от сида.
Трасса реально меняет гонку: грязный воздух/сопротивление обгону растут с
`1−overtaking`, клиппинг/расход — с `power`/`deploy`, износ — с `abrasion`+`downforce`.

## 12. Сейфти-кар

Отдельный поток событий `erng = RNG(mix32(seed))` — чтобы при `seed+1` (рестарт)
сценарий не повторялся. Розыгрыш один раз: если `erng.unit() < track.sc_prob`,
выезд на круге из окна `[SC_EARLIEST..SC_LATEST]` (0.15..0.70 дистанции). При выезде
пелотон **собирается в паровозик** (`SC_BUNCH_GAP`), темп падает (×1.40), питы
дешевле; уезжает через `SC_MIN..MAX_LAPS` (3..5).

## 13. Квалификация и старт (Волна 4)

**Квалификация:** один быстрый круг на seeded-RNG — время из скилла + характера
машины (`CAR_K`) + софт-компаунда + разброса ×(1−consistency) + редкого сорванного
круга (по composure). Сортировка по времени задаёт грид (`grid_pos`), машины
расставлены с шагом `GRID_GAP`. Силовая машина квалифицируется лучше на скоростных
трассах; апсеты случаются (отклонение от чисто-скилл-порядка ~1.6 мест).
**Старт:** разовый лонч на первом тике (`_race_start`) — дельта реакции из атрибута
`starts` + шум двигают `lap_frac` (хороший старт = +позиции). Полная Q1/Q2/Q3-сессия
и энерго-квала по секторам — в дизайне (UI-фаза), пока грид ставит этот лёгкий проход.

## 14. RNG и детерминизм

Два потока LCG: `rng` (потактовый темп/износ/питы/ошибки/DNF) и `erng` (события —
SC), сид которого прогнан через `mix32`. `gen_attributes` и генерация трасс — свои
короткоживущие RNG от сида. Один и тот же `seed_value` воспроизводит гонку
бит-в-бит → хост и клиент совпадают, харнесс повторяем.

## 15. Неткод (контекст)

Хост-авторитетный: только хост крутит `RaceSim`; клиент шлёт команды
(`set_pace`/`set_intent`/`request_pit`/`set_ers`/`set_overtake`) через `@rpc`, хост
рассылает снапшоты состояния, клиент их рендерит. Снапшот несёт `track`, цвета,
состояние машин (включая `dnf`) и флаг SC — карта у обоих совпадает.

## 16. Статус систем (engine v0.4) и что осталось

**Реализовано (самопроверено: Python-харнесс + `gdparse`/`gdlint` на экстрактах):**
- ✅ **Грязный воздух** (Волна 1): `_m_following` (`DA_*`) — потеря темпа в спутном
  потоке (трасса-зависимо) + харвест ×0.9; Overtake гасит.
- ✅ **Температура/прогрев шин** (Волна 1): `tyre_temp` с рабочим окном, холодный
  аутлэп → живой андеркат, перегрев → износ; `track_temp`/`air_temp` по трассам.
- ✅ **Персонал** (Волна 2, `personnel.gd`): роли+атрибуты; Стратег → качество пит-
  окна ИИ (разброс ×(1−S)); пит-экипаж → время/разброс/зевок стопа + надёжность.
- ✅ **Пит-лейн**: длина по трассам → потеря времени; визуальный заезд по пит-лейну.
- ✅ **Хаос** (Волна 3): лестница ошибок, контакты в борьбе (×1.6 у напарников),
  `aero_damage`→темп+ремонт, **причинный SC**, `pu_health`→рост DNF по дистанции.
- ✅ **Погода** (Волна 4): `wetness` на `erng`, inter/wet компаунды, `_m_weather`
  (кроссовер слик→инт ~0.4→дождь ~0.85), ИИ переобувается, дождь усиливает инциденты.
- ✅ **Квалификация + старт** (Волна 4): см. §13.

**Осталось (UI-фазы / мета — нужна итерация в Godot, харнессом не проверить):**
- **Скилы-инфо игрока** (Волна 2.3): туман телеметрии/прогнозов по скиллу инженера/
  стратега + богатые панели (ΔSoC / зона Overtake / кругов до клиффа).
- **Практики + сетап болида** (Волна 4.3): новая фаза уикенда, подбор идеала.
- **Полная Q1/Q2/Q3-сессия** + энерго-квала по секторам (сверх лёгкого прохода §13).
- **Кооп-кокпит**: разделить панели Директор/Инженер + RPC командных приказов.
- **R&D→болид** и **мета** (контракты/найм/паддок-политика): развитие `ENGINES`/`CHASSIS`.
- **Топливо как решение** (lift-and-coast), **обязательные два компаунда** — мелочи.
- **Энерго↔обгон**: харнесс не судит это взаимодействие — тюнить против живого движка.

## 17. Сводка тюнинг-констант (где что менять)

Тайминг `STEP`; характер машины `CAR_K`; сходы `DNF_BASE`; борьба `COMBAT_GAP`,
`MIN_GAP_S`, `PASS_DEADZONE`, `CREDIT_DECAY`, `_pass_resist`; энергия `CLIP_PENALTY`,
`OT_PACE/DRAIN/MIN_SOC/GAP_S`, `PASSIVE_REGEN`, `SOC_RECOVER`, `ERS_MODES`; шины
`COMPOUNDS`; темп `PACE_MODES`; SC `SC_*`; данные трасс `REAL_TRACKS`/`ARCHETYPES`;
болид `ENGINES`/`CHASSIS` (f1_2026). Все — `const` вверху файлов, баланс правится там.

## 18. Верификация

Python-харнесс (`simcheck.py`, `simcar*.py`, `ped2.py`, …) численно доказывает
правки (разрывы, андеркат, темп/ERS-трейдоффы, спред команд, характер машины, DNF,
частота SC). Синтаксис — `gdparse`/`gdlint` (gdtoolkit) на свежих extract-файлах
(обход обрезки маунта). Godot — финальный судья (запуск в песочнице невозможен).
