# AI Driver Brain v2 — дизайн (2026-06-11)

Синтез двух исследований: (1) как делают AI в гоночных/менеджерских играх
(Motorsport Manager, F1 Manager, Forza Drivatar, PURE GDC, iRacing, Football
Manager), (2) как реально ведут себя пилоты F1 (Verstappen/Hamilton/Alonso/
Perez/новички; ERS-поведение 2026). Полные отчёты с источниками — в истории
сессии; здесь — выжимка и архитектура для `race_sim.gd`.

## Принципы (несгораемые)

1. **AI платит ту же физику, что игрок.** Главная жалоба игроков на F1 Manager
   (4 релиза подряд): AI игнорирует деградацию шин/температуры и пушит 20
   кругов бесплатно. У нас AI уже едет по общей физике — это нельзя ломать.
2. **Никакого rubber-banding.** Видимое "подтягивание из ниоткуда" — самый
   ненавидимый паттерн. Если когда-нибудь понадобится catch-up — только PURE-
   модель (цель ±N м, кап, отключение на последних 20% гонки).
3. **Скилл ≠ поведение.** Два одинаково быстрых пилота должны *вести себя*
   по-разному (iRacing: Aggression/Optimism/Pit-Riskiness — отдельные ручки от
   скорости; FM: consistency/composure модулируют надёжность, не потолок).
4. **Детерминизм.** Все решения — чистая арифметика + существующие `rng`/`erng`.
   Никакого реального времени, никакой итерации по словарям в сим-пути.
5. **Не переусложнять.** Не нужен behavior tree / RL. Текущая rule-based схема
   + два новых скаляра состояния (patience, pressure) + мини-FSM атаки.

## Архитектура: три слоя

```
Execution  (каждый тик)   — темп/износ/ошибки; consistency/composure = дисперсия
Tactical   (каждый тик)   — pace mode × ERS mode × overtake: правила + новые скаляры
Combat     (по событию)   — pass credit (есть), + pressure, + pass-sticks bundle
Strategy   (по кругу)     — питы: undercut cover, 2-стоп план, SC-grab (есть)
```

## Новое состояние Driver

```gdscript
var attack_laps: int = 0      # кругов подряд в зоне атаки (<1.0s впереди)
var attack_backed: bool = false # отказался от атаки — «возьмём на питах»
var pressure: float = 0.0     # 0..1: накопленное давление (охотятся сзади)
var def_ers: bool = false     # тратит батарею на защиту (battery poker)
```

«Терпение» и «склонность пережимать» — производные, не хранятся:
- `patience_laps = 2 + race_iq*2 + composure*2` (≈2..6 кругов в зоне атаки)
- `overcommit = clampf(aggression - discipline, 0, 1)` (новички пережимают)

## Механики

### M-A1. Секторная атака (арминг по DRS-сектору)
Внутри OT_GAP, но текущий сектор без DRS → `ers_mode = "balanced"`,
`overtake = false` (копим SoC). Перед входом в DRS-сектор (sfrac > 0.8) и в
самом DRS-секторе → атака как сейчас. Убирает «горящее намерение» без эффекта
(сейчас на Монако 20 машин с overtake=true впустую). Зеркало реального
паттерна «deploy→close→harvest» из F1 Manager.

### M-A2. FSM атаки: терпение и отказ (AT-3 / AT-6, «Hamilton-режим»)
Считаем `attack_laps` (круги подряд в зоне <1.0s). Логика:
- `attack_laps <= patience_laps` → атакуем как сейчас (но по M-A1).
- Превысили терпение И credit не растёт (edge < deadzone стабильно) →
  `attack_backed = true`: садимся на 1.4с, `ers_mode` ⇒ harvest/balanced,
  pace ⇒ balanced, бережём шины. Радио: «не пройду — возьмём его на питах».
- Сброс `attack_backed`: цель запитовала / у нас свежее шины на 15+ ед.
  износа / цель совершила ошибку (pit_timer у неё > 0) / мы сами запитовали.
- Высокий aggression−discipline (overcommit) сокращает терпение и порог
  отказа — Verstappen-типы и новички лезут каждый круг.

### M-A3. Слив батареи на финише (RM-3)
`laps_left <= 3` → `floor_soc` линейно падает к ~12%. Все доезжают «в ноль»,
финальный спурт. Тривиально, реалистично.

### M-D1. Защитный расход батареи — «battery poker» (DF-3, 2026)
Реальная цитата Алонсо: «тратил энергию на прямых, чтобы у Чеко не было шанса».
2026-правила это усиливают: у лидера deploy гаснет к 355 км/ч, у атакующего
override до 337 — лидер обязан тратить раньше.
Реализация: защитник с угрозой сзади (<1.0s) и SoC > atk_off в DRS-секторе
ставит `def_ers = true` → расход SoC как attack, эффект: `+defending_bonus` к
порогу пасса (эквивалент −30..50% к credit-приросту атакующего, масштаб от
SoC). Кончилась батарея у защитника — дверь открыта. Симметричный ресурсный
покер вместо пассивного `defending * 3.5`.

### M-D2. Давление и вынужденная ошибка (PR-1/2/5/6)
`pressure` растёт, пока сзади сидят в <1.0s (≈ +0.10/круг, быстрее при
wetness), спадает со скоростью `composure` когда угроза ушла (>1.5s).
Эффекты:
- риск ошибки защитника: множитель `1 + pressure * (1.3 - composure)`
  в существующей формуле risk (race_sim.gd:1234) — классический lock-up
  в защите;
- лёгкий буст темпа защитника `−0.05с * pressure` (адреналин) — давление
  не только вредит, иногда мобилизует.
Асимметрия: атакующему давление не копится (охотнику нечего терять).

### M-C1. «Pass sticks» bundle (анти-пинг-понг, по F1 Manager)
После завершённого пасса:
- проигравший: `mood -= 0.2 + 0.2*(1-composure)`, его `pressure` обнуляется,
  3-4 тика без атаки (его credit уже 0 — достаточно короткого окна
  `attack_laps = -1` штрафа);
- победитель: 1 круг «counter-push» — floor_soc для атаки −8 (готов защитить
  позицию свежей батареей).
У нас пин-модель уже гасит пинг-понг; bundle делает это характерным, а не
механическим.

### M-S1. Undercut cover (реакция на пит соперника)
По событию «пит машины X»: каждый соперник в ±3.0с от X с `pit_count == 0`,
износом > 45 и laps_left > 8 с вероятностью `0.35 + 0.55*strat_skill`
(roll по `rng` один раз) помечает «cover» → питует в течение 1-2 кругов.
Слабый стратег прозевает undercut, сильный накроет.

### M-S2. Плановый 2-стоп на абразивных трассах
В `_strat_pit_wear`-момент (старт + после пита): если
`track.abrasion * COMPOUNDS[start].wear` прогнозирует >1.8 стинта — стратег с
`strat_skill > 0.55` выбирает план «2 стопа»: целевой износ −12, второй стоп
разрешён wear-триггером (`pit_count <= 1`). Разные трассы — разные гонки.

### M-S3. Gap-controller лидера (RM-1)
Лидер (P1, отрыв > 4с) при здоровых шинах → `conserve` + harvest до отрыва
2.5с; отрыв тает к 1.5с → возврат к balanced/push. «Управляет гонкой», а не
едет флэт-аут. Требует аккуратности: не должен делать лидера ловимым
бесплатно — кап на сэкономленный темп.

## Чего сознательно НЕ делаем

- Utility-scorer на 9 комбо-режимов каждый тик — overkill для наших 4 решений,
  правила + скаляры дают тот же результат дешевле и читаемее.
- Switchback/feint (AT-4/5) — микро-хореография не видна на нашем уровне
  абстракции (credit-модель), только усложнит.
- Rubber-banding в любом виде.
- Изменение `current_laptime()` — баланс-инвариант.

## Настройки болида + практика (добавлено 2026-06-11, по запросу владельца)

Пятая механика поверх 4 фаз. Скрытый идеал трассы
`track_ideal_setup(t)` = [0.10+downforce·0.80, 0.18+abrasion·0.35,
0.10+power·0.80] ± хэш-джиттер 0.08, кламп [0.05, 0.95].
`setup_q = 1 − L1/1.2` → `laptime += (1−q)·SETUP_PEN(0.45)` (гонка + обе
квалы). **Баланс-инвариант: базу строят инженеры по одной формуле для всех**
(`0.45 + 0.35·eng_telemetry + 0.15·race_iq ± 0.06`, кламп [0.35, 0.97]) —
игрок без практики едет на своей инженерной базе, как ИИ; практика
(`practice_run`: собственный hash-RNG, **не трогает rng гонки**) даёт
времена + покруговой фидбек пилота (точность = race_iq + telemetry,
коды 0/±1/±2/9). Зеркало: `car_setup_check.py`. Онлайн-клиент v1 — на базе
(практика клиенту — следующий шаг, через RPC сетапа как set_overtake).

## Принцип Фазы 2+ (директива владельца, 2026-06-11)

**Не гнаться за реализмом-2026** — сезон только начался, реальных метрик мало.
Приоритет — **баланс механик между ИИ и игроком**: любая смекалка, которую
получает ИИ-мозг, должна быть доступна через приказы игрока (иначе приказ —
ловушка). Аудит батарейки (Монца): клиппинг поля 3.7 % — здоров; поле живёт
на 22-27 % SoC, ~55 % круга в harvest — это identity Монцы, глобальные ставки
энергии не трогаем. Несимметрия была в приказах: «Атакуй» жёг 65 % attack-тиков
вне DRS (ИИ — 11 %) — исправлено в Фазе 2.

## Уроки Фаз 3-4 (классификация и сэндбокс MCP)

1. **`finish_time` ≠ классификация.** SC-физика догона (SC_CATCH_MULT) даёт
   отстающим быстрые круги — их «собственное» время дистанции меньше, чем у
   лидера. Классификация переведена на `finish_key` — суб-тиковый момент
   пересечения линии в общей шкале (`elapsed − overshoot·last_lt`);
   `finish_time` остался для гэпов в HUD.
2. **Сэндбокс `execute_gdscript` (godot MCP) фабрикует аномалии.** Внутри него
   `sim.order()` стабильно возвращал неотсортированный массив (3 разные
   реализации — один и тот же «мусор»), а чистый
   `godot --headless --script` дал 0 нарушений. Любую странность из MCP-прогона
   перепроверять чистым headless (`headless_suite.gd`). Балансовые счётчики
   (passes/dnf/pits) MCP, по сверке, отдаёт корректно.
3. **Плоский «−12» к цели износа не делает 2-стоп.** После раннего стопа AI
   ставил Hard до финиша и второй стоп не наступал. Работает: цель = деление
   гонки на ~3 стинта (`laps·wpl/3·1.15`, кламп [36, базовая]) + резина под
   стинт на первом стопе.

## Уроки Фазы 1 (что пришлось добавить против плана)

1. **Защёлка гистерезиса (`atk_latch`).** M-A1 сбрасывает ers_mode в
   non-DRS секторе → без защёлки порог входа (atk_on 46-56 %) применялся
   заново каждый круг, а на Монце SoC в борьбе ~22-26 % — атака не взводилась
   вообще (armed≈0, пассы 37→15). Латч держит бой «живым» (порог atk_off),
   сбрасывается при SoC ≤ atk_off (заряд броска истрачен) — без дребезга.
2. **Банковка = харвест, не balanced.** Balanced на Монце −24 %/круг — это
   не накопление. В бою вне DRS-зоны (или с неготовой батареей) — harvest.
3. **Сниженная планка для втянувшихся.** Пороги atk_on тюнились для круиза;
   бойцу в осаде (attack_laps ≥ 1) хватает atk_off+10 и окна 1.3×OT_GAP.
4. **Базис stall-edge: грязный воздух вычесть, слипстрим оставить.** «Чисто
   быстрее» — неверный критерий: на Монце обгоняют не более быстрые, а машины
   с подсосом. d.da_pen (вычитается — его отменяет Overtake) отделён от tow
   (остаётся). EMA 0.5 по кругам против шума одиночного тика.
5. **Доля «шумовых» пассов в старом коридоре.** ~46 % осаждающих на Монце
   реально медленнее лидера боя; их пассы в baseline были шумовыми («волшебные
   обгоны»). Новый коридор Монцы фактически 17-37 (без SC — нижний край).
