# Предложение: компонентная модель болида (CAR-1) — отложено («потом»)

Статус: **дизайн, не реализация.** Делать **после** того, как ablation-проверка
(`sim_ablation.gd`) подтвердит, что сим корректно учитывает 4–5 скаляров болида.

## Принцип (важно — архитектура)

Сим читает **скомпонованный** болид: `{power, aero, energy, reliability}` (+ harvest).
Это не меняем. Компоненты — **слой НАД композицией** (в `f1_2026.gd` / `season.gd`),
который собирает эти же скаляры. **Никаких новых входов в `race_sim.gd`.**

> Почему так: как только ablation докажет, что 4 скаляра симулируются верно, **любая**
> компонентная модель, питающая их, автоматически корректна. Детерминизм не затронут.

## Маппинг деталь → параметр болида

| Деталь | Влияет на |
|---|---|
| Переднее крыло / заднее крыло / днище / понтоны | **aero** |
| ДВС / турбо | **power** |
| Батарея / MGU-K / MGU-H | **energy** (harvest/deploy) |
| КПП / охлаждение / подвеска | **reliability** (+ малый aero/power) |

Каждая деталь: **уровень/спека** (качается через R&D — заменяет абстрактные «шаги»
aero/powertrain из META-1 на осязаемые детали) и опц. **состояние/износ** (деградирует
→ позже завязать на cost cap и штрафы за замену компонентов).

Композиция: итоговый скаляр = агрегат соответствующих деталей (напр. `aero =
base + Σ wing/floor levels`), затем `team_car()` отдаёт `{power, aero, energy, rel}`
как сейчас.

## Объём (прототип)

Не FM-уровень (десятки деталей, аэротруба, поставщики). **~6–8 деталей → 4 скаляра**,
уровень + состояние. Углублять позже.

## Зависимости / последовательность

1. Сначала: ablation-верификация скаляров (идёт сейчас).
2. Калибровка чего вне коридоров.
3. Потом: CAR-1 — компонентный слой над проверенными скалярами, завязанный на R&D
   (вытесняет абстрактные R&D-шаги META-1).
