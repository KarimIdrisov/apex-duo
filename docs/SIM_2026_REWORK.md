# Sim rework — overtaking calibration + 2026 energy/active-aero

Авторитетная запись по доведению гоночного ядра (`race_sim.gd`) до идеала на
**реальном движке** (Godot 4.6.3, headless). Всё проверено через `sim_bench.gd`
(SIM-VER), детерминизм сохранён, code-review: APPROVE-WITH-NITS (правки внесены).

## 1. Баг обгона (критично) — исправлено

Накопление «pass credit» интегрировалось **на тик** (`* dt`), а не на круг. При
~73 c/круг и шаге 0.25 c это **294 тика/круг** → кредит рос в **73–106× быстрее**
порога `_pass_resist`. Машина на 0.2 c/круг быстрее «продавливала» Монако меньше
чем за круг → **241 обгон/гонку** (цель ~1–6).

**Фикс:** `* dt` → `* (dt / track.base_laptime)` (как в соседнем расчёте контакта).
Кредит теперь = накопленный пейс-эдж в секундах, как и задумано. Монако **241 → 1.8**.

## 2. Плоский градиент → реалистичный (slipstream tow)

После фикса градиент стал плоским (~6 обгонов на всех трассах): сим моделировал
**грязный воздух (расталкивает)**, но не **слипстрим/тоу (стягивает)** — на силовых
трассах поле просто рассыпалось в чистом воздухе.

**Добавлено** в `_m_following`: **slipstream tow** — выигрыш темпа в более широкой
зоне (`SLIP_THRESH=1.4` c), гейт `max(0, power+overtaking−0.72)`, `SLIP_COEF=0.48`.
Плюс accrual кредита, масштабируемый `track.overtaking` (`PASS_OT_BASE=0.40`,
`PASS_OT_K=1.6`) — снижает обгон на низко-обгонных трассах, поднимает на высоких.
`_pass_resist`: `4.0+9.0·(1−ot)` → `3.0+8.0·(1−ot)`.

|---|---|---|---|
| Монако | 0.05 | 1.8 | 2.1 (нужен поул) |
| Хунгароринг | 0.12 | 3.3 | 2.6 |
| Сингапур | 0.18 | 2.8 | 3.3 |
| Зандворт | 0.20 | 10.8 | 3.2 |
| Сузука | 0.40 | 9.4 | 4.3 |

Монотонно по `track.overtaking`, и **правильно влияет на исход**: Монако — процессия

## 3. 2026 энергомодель / активное аэро — реворк (по реальным правилам)

Ресёрч правил 2026 (см. `docs/3D_VISUALIZATION_RESEARCH.md` нет — см. источники в
истории; FIA переименовала термины дек-2025, поправила энергию май-2026). Было:
один абстрактный SoC-«бак» на круг, без лимита энергии и без тейпера. Стало:

- **Бюджет энергии на круг** — `Track.energy_limit` (Монца 0.55 … Монако 1.00).
  Атака/Overtake тратят `deploy_budget` (`DEPLOY_BUDGET_BASE=8.5 × energy_limit`),
  сброс каждый круг; когда исчерпан — буст атаки гасится до конца круга.
- **Высокоскоростной тейпер** — выгода атаки `×(1 − TAPER_K·track.power)`
  (`TAPER_K=0.35`): на силовых трассах электробуст стоит меньше (энергия «теряется»
  выше ~290 км/ч).
- **Активное аэро — Straight Mode зоны** — `Track.aero_zones` (Монца/Баку 4 … Монако 0):
  выигрыш темпа `AERO_ZONE_K·zones·(0.5+0.5·car_power)` + лёгкая экономия SoC
  (`AERO_SOC_K`, только когда не в клиппинге).
- **Overtake Mode привязан к тейпер-дифференциалу** — `(0.5+0.5·power)·(0.4+0.15·zones)`:
  мощно на Монце/Баку, слабо на Монако.

**Проверка «батарея реально кусается» (реальный движок):** доля времени, когда
per-lap deploy budget исчерпан, корректно зависит от `energy_limit`:

| Трасса | energy_limit | deploy-limited % |
|---|---|---|
| Монако | 1.00 | 3.4% |
| Сузука | 0.85 | 14.8% |
| Сильверстоун | 0.80 | 16.0% |
| Монца | 0.55 | **37.4%** |

`clip%`≈0 теперь корректно: ограничитель сместился со «SoC-бак опустел» на
**per-lap бюджет** (верное по 2026). Машины финишируют с зарядом в запасе.

## Новые константы (`race_sim.gd`, top)
`SLIP_THRESH`, `SLIP_COEF`, `PASS_OT_BASE`, `PASS_OT_K`, `DEPLOY_BUDGET_BASE`,
`TAPER_K`, `AERO_ZONE_K`, `AERO_SOC_K`; новые поля `Track.energy_limit`,
`Track.aero_zones`, `Driver.deploy_budget`.

## SIM-VER — инструмент верификации
`sim_bench.gd` (headless SceneTree-скрипт). Запуск:
```
<godot> --headless --path ApexDuo_Prototype --script res://sim_bench.gd
```
Гоняет реальный `RaceSim` по всем трассам × N сидов, печатает коридоры обгона,
поул→победу, DNF, SC%, погоду, детерминизм. Это то, что Python-харнесс судить **не
может** (обгон/грязный воздух/погода/энергия). Держать как регресс-харнесс.
