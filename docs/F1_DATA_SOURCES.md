# Apex Duo — реальные данные F1 для калибровки симуляции

Ресёрч источников реальных данных F1 и **как они ложатся на тюнинг-константы движка**
(`race_sim.gd` / `f1_2026.gd`). Все данные — для калибровки чисел, не для геймплея в
рантайме (сим остаётся детерминированным и оффлайн). Лицензии у источников открытые
(API/библиотеки), но имена/логотипы F1 — торговые марки (см. ноту в `f1_2026.gd`).

## 1. Два главных конвейера данных

**Jolpica-F1 API** — поддерживаемый преемник закрытого в конце 2024 **Ergast API**
(тот обновлялся до конца сезона-2024 и отключён). Совместим с Ergast: меняешь
`ergast.com/api/f1/` на `api.jolpi.ca/ergast/f1/`. Покрытие — **все сезоны 1950–2026**,
есть **бесплатные CSV-дампы**. Лимит 200 запросов/час без ключа. Данные: результаты
(включая статус схода и причину), времена кругов, **пит-стопы**, квалификация,
зачёты, стартовые позиции.

**FastF1** (Python, MIT) — кэширует и парсит официальные тайминги/телеметрию F1.
Даёт по сессии: времена кругов и секторов, **компаунд и возраст шин (tyre life)**,
**пит-ин/пит-аут и номер стинта**, **погоду (track_temp, air_temp, влажность,
дождь/rainfall)**, телеметрию (скорость/газ/тормоз/передача/координаты), результаты.
Всё — Pandas DataFrame. Ключ не нужен. Это самый богатый источник для нашей модели.

## 2. Что чем калибровать (данные → константа сима)

| Реальные данные (источник) | Константа/система Apex Duo |
|---|---|
| FastF1: tyre life + темп круга по стинту | `COMPOUNDS` (`wear`/`cliff`), кривая деградации, прогрев `tyre_temp` |
| FastF1: **track_temp/air_temp/rainfall** по сессии | `Track.track_temp`/`air_temp` (`TRACK_TEMPS`), `wetness`, `wet_prob` |
| FastF1 / F1.com pit-summary: пит-ин−аут, потеря | `TRACK_PIT_LANES`, `PIT_BASE`/`PIT_LANE_K` |
| Jolpica: результаты со статусом DNF + причина | `DNF_BASE`, частоты инцидентов (`INCIDENT_K`/severity) |
| Jolpica: времена квалификации (гэпы P1→P20) | разброс квалы (`qnoise`), `CAR_K` |
| Jolpica: времена кругов | `base_laptime`, спред `skill` по гриду |
| StatsF1 / Lights-Out лог сейфти-каров | `sc_prob` по трассам, VSC-вариант |
| Pirelli: распределение компаундов C1–C5 по этапам | пер-трековый выбор `COMPOUNDS` |

## 3. Конкретные находки (и выводы для баланса)

- **Пит-стопы: ~20–25 c потери** на круг по трассам (Монца-2024 ≈ **23.7 c**). Наша
  модель `PIT_BASE 13 + lane×140` даёт 19–25.5 c — в диапазоне, но **Монца (19) занижена**
  против реальных ~23.7. Точные пер-трековые потери есть в pit-summary на F1.com → можно
  откалибровать `TRACK_PIT_LANES`.
- **Надёжность: современная F1 ~95%**, большинство сходов — **аварии, не техника**
  (Ред Булл за сезон ~3 схода, два — не по машине). У нас сейчас механических DNF ~2/гонку
  (`DNF_BASE`) + аварийные ~0.2 → перекос в сторону техники. **Вывод:** для реализма
  снизить `DNF_BASE` (механика ~0.5–1/гонку), а ретайрменты добирать инцидентами
  (Волна 3). Для «хаоса» можно оставить выше — это ползунок, теперь с реальной целью.
- **Сейфти-кар сильно зависит от трассы** (Канада — рекордсмен; был и 9-гоночный
  «сухой» отрезок без SC). Наш пер-трековый `sc_prob` — правильная модель; **VSC** реально
  есть (Австралия-2024 финишировала под VSC) → стоит добавить лёгкий VSC-вариант (в плане).
- **FastF1 отдаёт реальные track/air температуры по каждой сессии** → можно заменить наши
  прикидочные `TRACK_TEMPS` фактическими средними (и завести сезонную/погодную вариацию).

## 4. Как извлечь (набросок)

Калибровочные таблицы тянутся оффлайн в `outputs/` и сворачиваются в `const`-словари:

```python
# pip install fastf1 (и/или requests для Jolpica)
import fastf1
s = fastf1.get_session(2024, 'Monza', 'R'); s.load()
laps = s.laps                       # times, Compound, TyreLife, Stint, PitIn/OutTime
weather = s.weather_data            # AirTemp, TrackTemp, Rainfall, Humidity
# tyre deg: средний прирост темпа по TyreLife на компаунд → wear/cliff
# pit loss: медиана (PitOutTime − PitInTime) по этапу → TRACK_PIT_LANES
```

```text
# Jolpica (Ergast-совместимо): сходы и причины за сезон
GET https://api.jolpi.ca/ergast/f1/2024/results/   (статусы → доля механических DNF)
GET https://api.jolpi.ca/ergast/f1/2024/qualifying/ (гэпы P1→P20 → разброс квалы)
```

## 5. Приоритет применения

1. **Температуры трасс** из FastF1 → заменить прикидочные `TRACK_TEMPS` (дёшево, точнее).
2. **Пит-потери** из F1.com/FastF1 → подправить `TRACK_PIT_LANES` (Монца и др.).
3. **Доля механических DNF** из Jolpica → откалибровать `DNF_BASE` vs инциденты.
4. **Кривые деградации шин** из FastF1 tyre-life → уточнить `COMPOUNDS` и термомодель.
5. **Лог SC** → уточнить `sc_prob` и обосновать VSC.
