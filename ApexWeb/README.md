# Apex Web

Браузерный кооп-F1 в духе Sid Meier: два игрока — пилоты **одной команды** (две машины),
проходят уикенд на одной трассе (**Барселона**): 3 живые сессии Практики (P1/P2/P3) → живая Квала (нокаут Q1/Q2/Q3) → Гонка.
Гонка — реалтайм с синхронной паузой; остальные 20 машин грида (реальный сезон 2026) — AI.

Глубокий детерминированный сим (стиль Motorsport Manager / F1 Manager): топливо как жёсткий
ресурс + режим мотора (Save/Std/Push), резина с прогревом/кривой деградации/обрывом, 3 сектора
и 18 мини-секторов из реального контура трассы, слипстрим + грязный воздух, **боевые приказы
пилоту** (Атака/Защита — управляешь дуэлью ценой резины и риска локапа), **живые сейфти-кары от
реальных инцидентов** (0/1/2+ за гонку, завал в 1-м повороте), погода (сухо↔интер↔дождь),
**FM-модель команды** (13 атрибутов пилота вкл. квалу, 5 показателей болида, персонал: стратег +
пит-крю) и **ИИ-стратегия** (план питов, реакция на SC/дождь, управление мотором/темпом/приказом).
Выбор **сложности ИИ** (Лёгкая/Обычная/Сложная) в лобби. В гонке у игрока **4 рычага**: темп, мотор,
приказ (Атака/Защита) и пит.

Чистая статика: ES-модули + Canvas, без сборки. Онлайн — WebRTC P2P (PeerJS), хост-браузер
авторитетен (гоняет сим, клиент шлёт команды и рендерит снапшоты).

## Запуск локально

```
cd ApexWeb
python -m http.server 8000
```
Открой http://localhost:8000

**Редактирование трасс с сохранением в репо:** запусти вместо python мини-сервер:
```
cd ApexWeb
node tools/editor_server.mjs
```
Он отдаёт ту же статику (`http://localhost:8000`) **плюс** принимает Сохранить (пишет файл
`tracks/<slug>.json`) и Опубликовать (`git add ApexWeb/tracks && commit && push`). Без него
редактор работает, но Сохранить пишет только в localStorage.

**Разработка без интернета (две вкладки на одной машине):** в `src/ui/lobby.js` поставь
`const useP2P = false;` — транспорт переключится на `LocalNet` (BroadcastChannel между вкладками).
Открой две вкладки: в одной «Создать комнату», в другой «Войти».

## Онлайн (по умолчанию)

Хост жмёт **«Создать комнату»**, получает код и передаёт напарнику; тот вводит код и жмёт
**«Войти»**. Дальше каждый на своём экране настраивает свою машину; переход между фазами —
по кнопке «Готов» обоих (ready-gate). В гонке каждый крутит темп / режим мотора / пит своей машины.

## Практика — живые сессии (в духе F1 Manager)

Три сессии (P1/P2/P3) с **живым таймером**: пауза + ускорение ×2/×4/×8, либо «просимулировать
остаток» (быстрее, но результат хуже — руками выгоднее). Каждый игрок ведёт **свою машину**:
крутит **6 осей сетапа** (переднее/заднее крыло, подвеска, развал, передачи, тормозной баланс),
выбирает стинт (компаунд + круги) и **выпускает болид** — и каждый круг пилот копит **знание
трассы** (один показатель на пилота, держится через P1→P2→P3).

**Знание трассы — ядро глубины.** Оно **ограничивает точность сетапа**: пока трассу знаешь плохо,
**окна идеала** всех осей широкие и смещённые (фидбэк размытый: «мало кругов») — идеальный сетап
недостижим, хоть закатайся. С кругами знание растёт → окна **сужаются** к истине, фидбэк
конкретный («← чуть меньше», «нужно жёстче»), и сетап можно добить. Кривая реалистична:
**~60% сетапа за P1, ~80% за P2, ~95-100% за P3** — за одну сессию не выжать. Плюс знание трассы
даёт прямой **буст темпа** (уверенность пилота) в гонке — накатывать выгодно даже после
идеального сетапа. ИИ имеет базовое знание трассы, так что твоя практика — заработанное
преимущество, а не халява.

**Удовлетворённость %** (близость к идеалу) фиксируется только после реальных кругов на значении.
**Время боксов** на каждом выезде динамическое: смена шин (дороже при смене компаунда) + заправка
(×кругов в стинте) + применение изменённых настроек — часы сессии это бюджет, кнопка выезда
показывает разбивку «−Xс». Стинты дают кривую деградации (→ подсказка в гоночном HUD). Чёткость
фидбэка зависит от `race_iq` пилота. **Десктоп-раскладка:** крупный сетап слева, справа колонка —
метрики (знание трассы / удовлетворённость), выезд, стратегия, напарник; на узком экране в одну
колонку. Хост авторитетен (гоняет таймер сессии); каждый видит свою машину и напарника.

## Квала — нокаут Q1/Q2/Q3 (живая)

Полноценная F1-квала с живым таймером (пауза + ×2/×4/×8). Три сегмента с вылетами:
**Q1** (22 → вылетают 7) · **Q2** (15 → вылетают 5) · **Q3** (топ-10 за поул). Ты ведёшь свою
машину: **выпускаешь** её (свежий/тёплый софт) → out-lap греет шину → **быстрый круг едет вживую
по 3 секторам**. Видишь дельты по секторам и **выбираешь пуш** (сейв/норма/атака/предел), меняя его
по ходу круга. Жёстче = быстрее, но больше риск исполнения: **блокировка** (+время в секторе) или
**вылет** (круг **аннулируется**, заезд скрап). **Знание трассы из практики снижает риск** — кто
накатал, тот стабильно привозит круг; кто пропустил практику — ловит дикий разброс и теряет круги.
**«Сейв»** спасает круг, когда запахло аннуляцией. Можно 2-й круг на тёплом → in-lap; «прервать» в
любой момент.

Следишь за **тайминг-башней** с границей вылета. Четыре механики живого менеджмента: **эволюция
трассы** (резинится по ходу, поздние круги быстрее — но риск не успеть; ~1.3с за сессию), **лимит
софта** (~3 свежих комплекта), **трафик** (выезд в толпу компрометирует круг), **флаги** (красный
замораживает сессию и аннулирует круги — «банкер» спасает от вылета). 20 болидов грида хост
симулирует сам (ИИ тоже валит секторные круги под риском, в крайнем случае едет на «сейве» ради
времени); каждый игрок — своя машина, вылетевший наблюдает. **Десктоп-раскладка:** тайминг-башня
слева, карточка живого круга справа (дельты по секторам, контрол пуша, выезд). Сегменты стартуют на
паузе (▶). Итог сегментов → стартовый грид → гонка. Сетап заблокирован (parc fermé).

## Редактор трассы

`localhost:8000/editor.html` — конструктор трасс. Двигай опорные точки (дорога перерисовывается
живьём, как в игре), ставь объекты. **Колесо** — зум к курсору, **средняя кнопка** — пан, **⊡ по
размеру** — сброс вида. Режимы: **Точки** (форма + объекты), **Пит** (поставить боксы + потеря в
питах + рисуешь пит-лейн (вход/выход/сторона/ширина), по которому машины реально едут в гонке (2D+3D)), **Зоны** (на сетке 18 мини-секторов: красишь зоны обгона тормозн./слип + ease; ПКМ по
сектору — класс поворота · 🎯 Авто — расставить зоны по геометрии (правишь после)), **Старт**
(клик по трассе переносит линию старт/финиш в ближайшую опорную точку; «↺ Развернуть» — сменить
направление круга). **Сохранить** → трасса открывается в 3D-гонке; Экспорт/Импорт JSON.

**Пак трасс (общий через гит):** запусти редактор через `node tools/editor_server.mjs` — тогда
**Сохранить** пишет трассу файлом в `ApexWeb/tracks/<slug>.json` (и в localStorage), а **📤
Опубликовать** делает `git add ApexWeb/tracks && commit && push`. После пуша трасса грузится у всех
из папки `tracks/` (в редакторе — группа «Из репо», гонять — через 🏁). Без node-сервера Сохранить
остаётся только в localStorage.

**Превью рядом:** сверху панели — **3D-вид** (тот же движок, что в игре: трасса, сценерий, объекты и
пара машинок едут по линии; **🔄 Обновить 3D** перестраивает его под правки, мышь — орбита/зум) и
**svg-эталон** (оригинальный контур выбранной трассы для сравнения). 3D грузит THREE с CDN —
без сети превью отвалится с тостом, а 2D-редактор продолжит работать.

**🏁 Гонять (сим)** — гонка прямо на твоей трассе: её повороты/прямые определяют, где случаются
обгоны (слипстрим/грязный воздух + твои зоны), пит-потеря = твоё значение. Базовый темп круга пока
от Барселоны (геометрия влияет на бой, не на чистый темп — это следующий шаг).

## 3D-вид гонки (broadcast)

Полноэкранный 3D-вид (кнопка из гоночного HUD) переработан в духе Motorsport Manager — вся гонка
рисуется как телетрансляция:

- **Покрытие и поребрики.** Дорога — крашеная текстура (асфальт, кромки, разметка) с **гравийными
  ловушками и асфальтовыми зонами вылета снаружи поворотов**; поверх неё — **объёмные 3D-поребрики**
  (красно-белые, инстансингом) по обеим кромкам в поворотах. То же покрытие видит редактор трасс.
- **Окружение трассы** (детерминированно по имени трассы, модуль `scenery.js`): **трибуны с толпой**
  вдоль длинных прямых, **шинные барьеры** по внешней стороне поворотов, **рекламные щиты**,
  фоновые **деревья**, **посты маршалов** и **стартовая арка** над линией старт/финиш. Повторяющиеся
  объекты — через `InstancedMesh` (дёшево для отрисовки).
- **Болиды** — обновлённый силуэт F1: нос с законцовкой, понтоны, воздухозаборник, антикрылья,
  **halo**, шлем пилота в цвет команды, шины; кузов в цвете команды с лёгким блеском. **Боковина
  шин окрашена по компаунду** (soft красный / medium жёлтый / hard белый / inter зелёный / wet
  синий) и обновляется вживую при смене резины — стратегия читается прямо на трассе.
- **Камера: 3 режима** (кнопка «Камера»): **ТВ-режиссёр** (сам выбирает самую плотную дуэль на
  трассе и показывает её с разных трасс-камер, перекадровка каждые ~4–6 с), **обзор** (медленный
  облёт всей трассы + драг/зум), **погоня** за своей машиной / лидером с предиктивным взглядом вперёд.
- **Свет и атмосфера** — тёплый «золотой час» key-light + холодный fill, длинные тени, дымка-туман
  для глубины, ACES-тонмаппинг и мягкий bloom (с graceful-фолбэком, если CDN-постобработка не
  загрузилась). Всё — render-only поверх детерминированного сима: геймплей и нетворкинг не затронуты.
- **Погода в 3D** (по `snapshot.wetness` из сима): мокрый асфальт **блестит** (роняем roughness,
  поднимаем metalness → блики от солнца), небо/туман **сереют**, солнце приглушается, идёт **дождь**
  (частицы), а из-под колёс летят **брызги** (полупрозрачный шлейф, интенсивность по влажности).
  Сухо ↔ дождь меняются плавно вслед за погодной моделью сима.

- **Паритет с MM (2026-06):** над каждым болидом — **плавающая бирка** «P{позиция} {код пилота}»
  в цвете команды (спрайт-биллборд, обновляется вживую при смене позиции); на полотне — **чёткие
  белые кромки** и **«резиновая» гоночная линия** по центру; старт-финиш **обжит** (пит-билдинг с
  боксами + плотные трибуны); модели болидов **сглажены** (капсульные корпус/понтоны + конусный нос
  вместо «коробок»); масштаб трассы увеличен (WORLD 175) — круг крупнее, машины читаются мельче.
- **Болид-модель v2 + анти-наложение (2026-06):** болид собран как узнаваемый F1 — скруглённый
  корпус-капсула, конусный нос, **многоэлементные** переднее/заднее крылья с законцовками, кокпит
  с halo и шлемом, хромированные ступицы, цветные обода шин; кузов с **глянцевым лаком**
  (`MeshPhysicalMaterial` clearcoat). В кадровом цикле — **боковое расталкивание** (render-only):
  близкие по дистанции болиды разъезжаются по полотну и едут бок-о-бок, а не сквозь друг друга
  (плюс убирает дёрганье от «толкотни»); постановка сглажена двойным лоу-пассом.

**Быстрый просмотр без уикенда:** открой `http://localhost:8000/showroom.html` — автономный
**3D-шоурум**: полный грид из 22 машин едет по сцене, переключение камер/трасс «на лету» и кнопка
дождя.

## Структура

```
index.html      каркас + PeerJS CDN + точка входа
src/rng.js      детерминированный RNG (LCG + mix32)
src/data.js     грид 2026, трасса, компаунды/режимы, FM-показатели, сложность, тюнинг-константы
src/sim.js      чистый детерминированный сим (class Race): темп/резина/топливо, бой + боевые приказы (Атака/Защита), питы, живые SC от инцидентов, погода, DNF, грид. **Штрафы стюардов:** дерзкий бол-обгон вне зоны может схватить **5с тайм-пенальти за контакт** (`penaltyTimer` — теряешь время на трассе, как мини-стоп; событие `penalty` → лента). **Брак пит-стопа:** уставший/несыгранный пит-крю мажет на стопе (см. `pitcrew.js`) — событие `pit{mishap}` кормит ленту. **Старт:** существующая launch-модель теперь эмитит события хорошего/плохого старта (`launch_good`/`launch_bad`) для комментария
src/fuel.js     топливо как ресурс: режимы мотора, вес, расход, fuelLaps
src/tyres.js    прогрев (холодный выезд) + кривая деградации + обрыв
src/track.js    3 сектора / 18 мини-секторов из контура, sampleAt, miniSplits
src/geom3d.js   геометрия 3D-вида (чистая): осевая, лента (кривизна-кламп → без самопересечения в поворотах), линия гонки, камера, кербы-маска
src/overtake.js слипстрим, грязный воздух (износ), накопление пасс-кредита
src/events.js   инцидент-хелперы (incidentChance / cautionFromIncident) → живые SC/VSC от аварий
src/weather.js  таймлайн дождя, кроссовер слик↔интер↔дождь + **прогноз-радар (`liveForecast`):** из скрытой дуги дождя на текущем круге выводит игроку прогноз с УВЕРЕННОСТЬЮ, растущей по мере приближения фронта — далёкий → «переменно», близкий → точное окно ETA и %; во время дождя — оценка «подсохнет через ~N кругов». Это основа ставки слик↔интер↔дождь. В HUD гонки — цветная полоса-радар (надвигается/сыро/дождь/подсыхает), в паддоке на карточке этапа — пред-гоночный прогноз по климатической вероятности трассы. Детерминированно
src/staff.js    штаб + **рефактор «Команда» (фаза 1, 2026-06):** специальности теперь дают КОНКРЕТНЫЙ эффект (`SPECIALTIES.fx`/`specialtyBonus`): аэродинамик → +6% разработка (`devMult`), механик → +надёжность машины (`staffRelBonus`→`applyRaceMods`), тактик → +стратегия (`composePersonnel`), ас пит-стопа → быстрее стопы (`pitMult`). Вкладка переразбита на **«Штаб»**: департаменты карточками (Конструктор/Стратег/Пит-крю + КБ/Пит-бокс/Завод) с эффектом числом и одной кнопкой; рынок специалистов компактный с фильтром по роли; усталость/содержание/зарплаты сверху. **Фаза 2 (живой штаб, v24):** **именованный штаб у всех 11 команд** (`initTeamStaff`, вымышленные имена, рейтинг по силе команды); **рынок = свободные агенты + штаб соперников** (`staffMarketAll`), фильтр по роли — **переманивание** ривала (`hireFromMarket`) с премией к гонорару, при этом соперник теряет специалиста (замена слабее + −gridBoost); **контракты штаба** тикают по сезонам, в межсезонье соперник может **переманить твою звезду** с истёкшим контрактом (удерживай продлением `reSignStaff`); **развитие персонала** (`tickStaffTrain` — роль в обучении растёт по гонкам, КБ ускоряет, малая плата); **стройка как проект** (`startFacilityProject`/`tickFacility` — апгрейд завода/КБ/пит-бокса идёт днями по календарю). Схема сейва v24
src/drivers.js  модель пилота + **углубление (G1–G4, см. docs/DRIVERS_DEPTH.md, схема v23):** статистика сезона + дуэль с напарником (квала/гонка H2H), тренировка с фокусом (выбранные навыки растут быстрее в сезон+зиму, стрелки развития), мораль от формы/напарника/контракта с причиной + запросы пилота (контракт / статус 1-го номера, принять/отклонить), развитие черт (мастерство атрибута → новая черта). Богатая карточка во вкладке «Пилоты»: форма/мораль, контракт, статистика, дуэль, селектор тренировки, бар-чарт 13 атрибутов со стрелками. **Старение и уход (`retirePass`):** возрастная кривая уже сводит атрибуты ветеранов вниз через `attrDrift`; в межсезонье `retireChance(age,overall)` отправляет ветеранов на пенсию (с 34 лет, шанс растёт с возрастом, элитный овр продлевает карьеру), место занимает дебютант из `ROOKIE_POOL` (грид всегда 22) — освобождаются боевые места и рождается спрос на выпускников академии. Карточка пилота показывает тренд возраста (↗ растёт / ↘ ветеран); лента новостей — уходы и дебюты. Детерминированно. **Контрактные требования по ходу сезона (`makeDriverRequest`):** пилот в форме просит по ситуации — статус №1, **вписать бонусы за результат** в контракт (выигрывает гонки), **прибавку к зарплате** (серия подиумов) или продление; принятие даёт мораль, но стоит дороже потом (бонусы платятся по результатам, прибавка +25% каждую гонку), отказ бьёт по морали сильнее у звёзд. Панель запроса в карточке «Пилоты» с подсказкой о цене
src/market.js   трансферы + **рефактор «Трансферы» (2026-06):** богатые карточки рынка (аватар/команда/OVR/возраст/навыки/разбивка цены/бейдж «свободен»), **индикатор интереса** пилота к твоей команде (`interest` — звезда не пойдёт в слабую), фильтр (свободные/OVR/цена/возраст), **переговоры по контракту** (длина 1/2/3 → выше согласие, но дороже, `signCostAt`/`negotiateSign(length)`), **лента слухов** (свободные агенты, движения, интерес соперников к твоим пилотам `rivalInterest`); риск реальный — в межсезонье соперник может **подписать твою звезду** с истёкшим контрактом (удерживай продлением). **Глубокие контракты (`CLAUSE`/`buildClauses`):** при подписании выбираешь клаусулы — **бонусы за результат** (подиум/победа/титул выплачиваются из бюджета в `applyResult`/конце сезона, взамен дешевле аванс), **гарантия №1** (пилот охотнее переходит, получает статус первого номера), **пункт выкупа** (дешевле сейчас, но соперник может выкупить из контракта — повышает риск ухода в межсезонье и в ленте слухов). Тумблеры в контролах «Трансферов», активные клаусулы — чипами на карточке пилота. **Агент пилота с торгом (`buildCounter`/`applyCounter`):** пограничный пилот вместо мгновенного отказа выдаёт **контрпредложение** (выше гонорар +20% / контракт на 3 сезона / гарантия №1) — баннер на карточке с «Принять условия» / «Отказаться»; принятие форсирует сделку на условиях агента. Совсем недосягаемая звезда в слабой команде по-прежнему отказывает наотрез. Детерминированно (`career.negotiation` + команды `career_sign_accept`/`career_sign_cancel`)
src/academy.js  **академия как пайплайн талантов (v2, схема сейва v26):** лестница серий **Ф4→Ф3→Ф2→Ф1** — каждое межсезонье юниоры едут детерминированный фидер-сезон в своей серии, копят **очки суперлицензии** (реалистичная FIA-таблица по серии и месту, скользящее окно 3 сезона, гейт 40), развиваются к **скрытому потенциалу**. **Скаутинг** раскрывает потолок постепенно: широкий диапазон + звёзды уверенности (низкая уверенность может ввести в заблуждение), `scoutProspect` за $ сужает оценку. **Контракты + переманивание** — незаконтрактованных ярких юниоров уводят соперники (`developAcademy` → лента слухов). **Роли юниора:** резерв (R&D-бонус), **FP1-программа** (+опыт/СЛ), **аренда в боевой кокпит соперника** (опыт + гонорар $0.6M, риск выкупа), **промоушн** в свой состав (гейт по суперлицензии). **Программа академии** (`academyTier` 0–5, buy-vs-wait) ускоряет развитие, удешевляет скаутинг, открывает слоты и ярче таланты в пуле. Градация — **максимум одна ступень за сезон** (отложенная, чтобы юниор не перепрыгивал серии). Богатая вкладка «Академия»: header программы, пайплайн-лестница, карточки юниоров (диапазон потолка со звёздами, прогресс СЛ до 40, бар развития, архетип, контракт, кнопки ролей/аренды/промоушна), фидер-таблицы по сериям, скаутинг-карточки, новости. **Личности (`PERSONA`):** у юниора характер (преданный/наёмник/вспыльчивый/амбициозный) + мораль — влияют на шанс переманивания (преданный держится, наёмник уходит легко), цену продления (`extendCost`), скорость развития (мораль ×0.85..1.15) и поведение (вспыльчивый теряет мораль без игрового времени, амбициозный рвётся в Ф1 со скамейки). Чип персоны на карточке и в тултипе; тултип юниора — отдельный (серия-цвет + биас атрибутов под архетип + бар морали). **Соперники-академии (`rivalCourting`/`career.rivalJuniors`):** конкурирующие академии уводят ярких проспектов из открытого пула каждое межсезонье (до 2 за зиму, шанс ∝ потенциалу) — скаутинг стал срочным, на карточке скаутинга яркий проспект помечается «🔥 соперники присматриваются». **Хроника выпускников (`career.graduates`):** промоутнутый юниор помечается `fromAcademy`, и каждое межсезонье его сезон банкуется в карьерную запись (сезоны/победы/подиумы/очки/титулы/команда, чемпион → титул, ушедшие помечаются неактивными) — панель «🎓 Выпускники академии» на вкладке с их историей в Ф1. Платёж = эмоциональная отдача от вложений в пайплайн. Влияние на сим — только через `academyDevBonus()`. Детерминировано (mix32), кооп-безопасно; проверка node-сниппетами (academy/personas/retirement/rival-academies)
src/pitcrew.js  **пит-крю как управляемый юнит:** 5 ролей (передний/задний домкрат, гайковёрт, колёсный, стоп-мен), именованные механики со скиллом/усталостью/травмами+дублёрами и **сыгранностью** (cohesion). Композится в `pitMult` (скорость стопа) + шанс **брака/катастрофы** стопа (`composePitCrew`). **Тренировка** (растит скилл весь сезон ценой усталости), **практика пит-стопов** (разовый буст скилла+сыгранности, утомляет), **рынок механиков** (нанять в роль — новая химия сбрасывает cohesion). Усталость копится за гонки, восстанавливается зимой; вымотанный крю рискует травмой в межсезонье (выходит дублёр −0.18). В гонке (`sim.js`) уставший/несыгранный крю **чаще мажет на стопе** — медленный стоп (+1.5–4с) или редкая катастрофа (+8–14с, небезопасный выпуск). Панель «Пит-крю» в «Команде»: карточки ролей, метрики скорости/сыгранности/усталости/риска, кнопки тренировки/практики, рынок. Детерминировано, кооп-безопасно; проверка node-сниппетом

src/backers.js     архетипы финансирования команд (works/independent + годовой грант + свой/клиентский ДВС) по гриду 2026; см. docs/FINANCE_TEAMS_DESIGN.md. Фаза A: грант = доход-пол за этап, сезонный призовой фонд сплющен до ~2.3× (как реальная F1, анти-отрыв); дашборд «Финансы» показывает архетип/грант. **Фаза B — программы ДВС** (`development.js`): у заводских — меню разработки силовой установки (ветки Мощность/Эффективность/Надёжность, **ВНЕ кост-капа** — реалистичная асимметрия) + доход с поставки ДВС клиентам; у клиентских — путь «свой ДВС» (сделка с концерном: полная программа или бейдж-партнёрство, многосезонная постройка → становишься заводской PU-командой) + плата за клиентский ДВС. ДВС композится в боевую машину игрока через `effectiveCarPU`. **Фаза C — события** (`team_events.js`): в межсезонье возможен **выкуп команды** суитором (автоконцерн/техногигант/фонд; интерес зависит от результата+независимости) — принять (с выбором: сохранить имя или **ребрендинг** имя+цвет) даёт кэш+грант (+свой ДВС от концерна) и новую цель совета, отклонить — сохраняешь независимость; **уход концерна** у проваливших заводских (грант срезан, ДВС → клиентский); **живой грид** (соперника выкупают и усиливают, `gridBoost`); редкие **денежные события** за этап (спонсор/штраф/вливание). Детерминированно, кооп-безопасно. Схема сейва v16. **Фаза D — баланс (честность игрок↔ИИ):** реактивный ИИ — на высокой сложности соперник **прикрывает андеркат** (упреждающий пит при близкой угрозе сзади у окна стопа; `ai_strategy.pitDecision` reason `cover`, порог difficulty>0.7), на лёгкой не реагирует → обыгрываем; **редкие стратегические ошибки** ИИ на лёгкой/обычной (грубо мимо окна стопа, `planRace` blunder ∝ 1−difficulty) → низкая сложность бьётся смекалкой. Замер потолка скилла: perfect-практика даёт игроку лишь ~0.06 c/круг над базой ИИ (setup −0.15 vs ~−0.11, трек-нолидж −0.08 vs ×0.7) — на чистом темпе сетап «перевозит» мало (реалистично), главный рычаг скилла — стратегия/квала, откалиброванная реактивным ИИ.
```
