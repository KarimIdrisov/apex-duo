# Apex Web

Браузерный кооп-F1 в духе Sid Meier: два игрока — пилоты **одной команды** (две машины),
проходят уикенд на одной трассе (**Барселона**): 3 живые сессии Практики (P1/P2/P3) → живая Квала (нокаут Q1/Q2/Q3) → Гонка.
Гонка — реалтайм с синхронной паузой; остальные 20 машин грида (реальный сезон 2026) — AI.

Глубокий детерминированный сим (стиль Motorsport Manager / F1 Manager): топливо как жёсткий
ресурс + режим мотора (Save/Std/Push), резина с прогревом/кривой деградации/обрывом, 3 сектора
и 18 мини-секторов из реального контура трассы, слипстрим + грязный воздух, сейфти-кар, погода
(сухо↔интер↔дождь), **FM-модель команды** (13 атрибутов пилота вкл. квалу, 5 показателей болида,
персонал: стратег + пит-крю) и **ИИ-стратегия** (план питов, реакция на SC/дождь, управление
мотором/темпом). Выбор **сложности ИИ** (Лёгкая/Обычная/Сложная) в лобби.

Чистая статика: ES-модули + Canvas, без сборки. Онлайн — WebRTC P2P (PeerJS), хост-браузер
авторитетен (гоняет сим, клиент шлёт команды и рендерит снапшоты).

## Запуск локально

```
cd ApexWeb
python -m http.server 8000
```
Открой http://localhost:8000

**Разработка без интернета (две вкладки на одной машине):** в `src/ui/lobby.js` поставь
`const useP2P = false;` — транспорт переключится на `LocalNet` (BroadcastChannel между вкладками).
Открой две вкладки: в одной «Создать комнату», в другой «Войти».

## Онлайн (по умолчанию)

Хост жмёт **«Создать комнату»**, получает код и передаёт напарнику; тот вводит код и жмёт
**«Войти»**. Дальше каждый на своём экране настраивает свою машину; переход между фазами —
по кнопке «Готов» обоих (ready-gate). В гонке каждый крутит темп / режим мотора / пит своей машины.

## Практика — живые сессии (в духе F1 Manager)

Три сессии (P1/P2/P3) с **живым таймером**: пауза + ускорение ×2/×4/×8, либо «просимулировать
остаток» (быстрее, но результат хуже — руками выгоднее). Каждый игрок ведёт **свою машину**:
крутит **6 осей сетапа** (переднее/заднее крыло, подвеска, развал, передачи, тормозной баланс),
выбирает стинт (компаунд + круги) и **выпускает болид** — и каждый круг пилот копит **знание**.
У каждой оси скрытый оптимум: пока знания мало, **окно идеала** широкое, а фидбэк размытый
(«мало кругов»); с кругами окно **сужается** к истине, фидбэк становится конкретным
(«← чуть меньше», «нужно жёстче»). Двигаешь ползунок в окно → характеристика «оптимально».
**Удовлетворённость %** фиксируется только после реальных кругов на значении и даёт буст темпа
в квале и гонке. Знание переносится P1→P2→P3; стинты заодно дают кривую деградации (данные
стратегии → подсказка в гоночном HUD). Чёткость фидбэка зависит от `race_iq` пилота; скрытый
оптимум у двух пилотов слегка разный. Хост авторитетен (гоняет таймер сессии); каждый видит свою
машину и удовлетворённость напарника. Каждый выезд тратит **время боксов** (смена шин + заправка
+ применение изменённых настроек, пропорц. размаху правок) из часов сессии — крутить бесконечно
не выйдет, часы — это бюджет.

## Квала — нокаут Q1/Q2/Q3 (живая)

Полноценная F1-квала с живым таймером (пауза + ×2/×4/×8). Три сегмента с вылетами:
**Q1** (22 → вылетают 7) · **Q2** (15 → вылетают 5) · **Q3** (топ-10 за поул). Ты ведёшь свою
машину: **выпускаешь** её (свежий/тёплый софт + спокойно/атака) → out-lap греет шину → **быстрый
круг** засчитан → можно 2-й на тёплом → in-lap; «прервать» в любой момент. Следишь за
**тайминг-башней** с границей вылета. Четыре механики живого менеджмента: **эволюция трассы**
(резинится по ходу, поздние круги быстрее — но риск не успеть; ~1.3с за сессию), **лимит софта**
(~3 свежих комплекта), **трафик** (выезд в толпу компрометирует круг), **флаги** (красный
замораживает сессию и аннулирует круги — «банкер» спасает от вылета). 20 болидов грида хост
симулирует сам; каждый игрок — своя машина, вылетевший наблюдает. Сегменты стартуют на паузе
(спланируй выезд → нажми ▶). Итог сегментов → стартовый грид → гонка. Сетап заблокирован (parc fermé).

## Редактор трассы

`localhost:8000/editor.html` — двигай опорные точки трассы (дорога перерисовывается живьём,
точно как в игре), ставь объекты (трибуны/баннеры/деревья/конусы), **Сохранить** → трасса
открывается в 3D-гонке. Экспорт/Импорт JSON для бэкапа. Хранится в localStorage (на браузер).

## Тесты и баланс

```
cd ApexWeb
node --test                 # модульные тесты чистого ядра (160: rng/data/sim/fuel/tyres/track/overtake/events/weather/team/ai_strategy/setup/practice/practice_session/quali/quali_session/weekend/net; sim.test.js гоняет сотни полных гонок — ~10 мин)
node tools/balance.mjs      # балансные коридоры (DNF, разброс, питы/топливо, сектора, обгоны, SC, погода, атрибуты, стратегия, сложность)
```

## Структура

```
index.html      каркас + PeerJS CDN + точка входа
src/rng.js      детерминированный RNG (LCG + mix32)
src/data.js     грид 2026, трасса, компаунды/режимы, FM-показатели, сложность, тюнинг-константы
src/sim.js      чистый детерминированный сим (class Race): темп/резина/топливо, бой, питы, SC, погода, DNF, грид
src/fuel.js     топливо как ресурс: режимы мотора, вес, расход, fuelLaps
src/tyres.js    прогрев (холодный выезд) + кривая деградации + обрыв
src/track.js    3 сектора / 18 мини-секторов из контура, sampleAt, miniSplits
src/geom3d.js   геометрия 3D-вида (чистая): осевая, лента (кривизна-кламп → без самопересечения в поворотах), линия гонки, камера, кербы-маска
src/overtake.js слипстрим, грязный воздух (износ), накопление пасс-кредита
src/events.js   расписание сейфти-кара + старт-инциденты
src/weather.js  таймлайн дождя, кроссовер слик↔интер↔дождь
src/team.js     FM-модель: 13 атрибутов пилота, 5 показателей болида, персонал
src/ai_strategy.js  ИИ: план питов, решение о пите (погода/SC/план), режим мотора/темпа
src/setup.js    6 осей: скрытый идеал (трасса+пилот), сужающееся окно знания, фидбэк, удовлетворённость
src/practice.js чистые хелперы практики: carMean + practiceLapBase (база темпа круга)
src/practice_session.js  живая сессия (чистая, детерм.): таймер/скорость, накопление знания за круг, окно/фидбэк, авто-сим, снапшот
src/quali.js    qualiLap: круг квалы (атрибут квалы + grip/шина/трафик/жёлтый) + buildGrid (фолбэк-решётка)
src/quali_session.js  живая квала (чистая, детерм.): нокаут Q1/Q2/Q3, цикл out-lap→flying→in-lap, эволюция трассы, шины/трафик/флаги, ИИ-выезды, тайминг-башня, finalGrid
src/weekend.js  машина состояний уикенда + ready-gate
src/net.js      транспорт: LocalNet (вкладки) + P2PNet (PeerJS)
src/main.js     склейка: хост-цикл, команды/снапшоты, выбор экрана
src/ui/*.js     экраны: lobby (команда + сложность), practice (живая сессия: таймер, 6-осевой виджет знания, питчер стинтов, панель деградации), quali (живая тайминг-башня: сегменты Q1/Q2/Q3, эволюция/шины/трафик/флаги), race (HUD + мини-карта + полноэкранный 3D-вид: трасса с кербами-в-поворотах, болиды, камера, тон-маппинг/тени/блум)
tools/balance.mjs  балансный харнесс
editor.html        автономная страница редактора трассы
src/ui/editor.js   редактор трассы (тащи точки, ставь объекты, экспорт/импорт JSON)
src/track_paint.js общая отрисовка трассы на Canvas (редактор + 3D-вид)
src/track_store.js localStorage-хранилище отредактированных трасс
```

Деплой на GitHub Pages — через `.github/workflows/pages.yml` (push в `main`, изменения в `ApexWeb/`).
В настройках репозитория Settings → Pages выбери источник **GitHub Actions**.
