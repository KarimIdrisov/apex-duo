# Apex Web

Браузерный кооп-F1 в духе Sid Meier: два игрока — пилоты **одной команды** (две машины),
проходят уикенд на одной трассе (**Барселона**): 3 живые сессии Практики (P1/P2/P3) → живая Квала (нокаут Q1/Q2/Q3) → Гонка.
Гонка — реалтайм с синхронной паузой; остальные 20 машин грида (реальный сезон 2026) — AI.

Глубокий детерминированный сим (стиль Motorsport Manager / F1 Manager): топливо как жёсткий
ресурс + режим мотора (Save/Std/Push), резина с прогревом/кривой деградации/обрывом, 3 сектора
и 18 мини-секторов из реального контура трассы, слипстрим + грязный воздух, **боевые приказы
пилоту** (Атака/Защита — управляешь дуэлью ценой резины и риска локапа), **живые сейфти-кары от
реальных инцидентов** (0/1/2+ за гонку, завал в 1-м повороте), погода (сухо↔интер↔дождь),
**FM-модель команды** (13 атрибутов пилота вкл. квалу, 5 показателей болида, персонал: стратег +
пит-крю) и **ИИ-стратегия** (план питов, реакция на SC/дождь, управление мотором/темпом/приказом).
Выбор **сложности ИИ** (Лёгкая/Обычная/Сложная) в лобби. В гонке у игрока **4 рычага**: темп, мотор,
приказ (Атака/Защита) и пит.

Чистая статика: ES-модули + Canvas, без сборки. Онлайн — WebRTC P2P (PeerJS), хост-браузер
авторитетен (гоняет сим, клиент шлёт команды и рендерит снапшоты).

## Запуск локально

```
cd ApexWeb
python -m http.server 8000
```
Открой http://localhost:8000

**Редактирование трасс с сохранением в репо:** запусти вместо python мини-сервер:
```
cd ApexWeb
node tools/editor_server.mjs
```
Он отдаёт ту же статику (`http://localhost:8000`) **плюс** принимает Сохранить (пишет файл
`tracks/<slug>.json`) и Опубликовать (`git add ApexWeb/tracks && commit && push`). Без него
редактор работает, но Сохранить пишет только в localStorage.

**Разработка без интернета (две вкладки на одной машине):** в `src/ui/lobby.js` поставь
`const useP2P = false;` — транспорт переключится на `LocalNet` (BroadcastChannel между вкладками).
Открой две вкладки: в одной «Создать комнату», в другой «Войти».

## Онлайн (по умолчанию)

Хост жмёт **«Создать комнату»**, получает код и передаёт напарнику; тот вводит код и жмёт
**«Войти»**. Дальше каждый на своём экране настраивает свою машину; переход между фазами —
по кнопке «Готов» обоих (ready-gate). В гонке каждый крутит темп / режим мотора / пит своей машины.

## Практика — живые сессии (в духе F1 Manager)

Три сессии (P1/P2/P3) с **живым таймером**: пауза + ускорение ×2/×4/×8, либо «просимулировать
остаток» (быстрее, но результат хуже — руками выгоднее). Каждый игрок ведёт **свою машину**:
крутит **6 осей сетапа** (переднее/заднее крыло, подвеска, развал, передачи, тормозной баланс),
выбирает стинт (компаунд + круги) и **выпускает болид** — и каждый круг пилот копит **знание
трассы** (один показатель на пилота, держится через P1→P2→P3).

**Знание трассы — ядро глубины.** Оно **ограничивает точность сетапа**: пока трассу знаешь плохо,
**окна идеала** всех осей широкие и смещённые (фидбэк размытый: «мало кругов») — идеальный сетап
недостижим, хоть закатайся. С кругами знание растёт → окна **сужаются** к истине, фидбэк
конкретный («← чуть меньше», «нужно жёстче»), и сетап можно добить. Кривая реалистична:
**~60% сетапа за P1, ~80% за P2, ~95-100% за P3** — за одну сессию не выжать. Плюс знание трассы
даёт прямой **буст темпа** (уверенность пилота) в гонке — накатывать выгодно даже после
идеального сетапа. ИИ имеет базовое знание трассы, так что твоя практика — заработанное
преимущество, а не халява.

**Удовлетворённость %** (близость к идеалу) фиксируется только после реальных кругов на значении.
**Время боксов** на каждом выезде динамическое: смена шин (дороже при смене компаунда) + заправка
(×кругов в стинте) + применение изменённых настроек — часы сессии это бюджет, кнопка выезда
показывает разбивку «−Xс». Стинты дают кривую деградации (→ подсказка в гоночном HUD). Чёткость
фидбэка зависит от `race_iq` пилота. **Десктоп-раскладка:** крупный сетап слева, справа колонка —
метрики (знание трассы / удовлетворённость), выезд, стратегия, напарник; на узком экране в одну
колонку. Хост авторитетен (гоняет таймер сессии); каждый видит свою машину и напарника.

## Квала — нокаут Q1/Q2/Q3 (живая)

Полноценная F1-квала с живым таймером (пауза + ×2/×4/×8). Три сегмента с вылетами:
**Q1** (22 → вылетают 7) · **Q2** (15 → вылетают 5) · **Q3** (топ-10 за поул). Ты ведёшь свою
машину: **выпускаешь** её (свежий/тёплый софт) → out-lap греет шину → **быстрый круг едет вживую
по 3 секторам**. Видишь дельты по секторам и **выбираешь пуш** (сейв/норма/атака/предел), меняя его
по ходу круга. Жёстче = быстрее, но больше риск исполнения: **блокировка** (+время в секторе) или
**вылет** (круг **аннулируется**, заезд скрап). **Знание трассы из практики снижает риск** — кто
накатал, тот стабильно привозит круг; кто пропустил практику — ловит дикий разброс и теряет круги.
**«Сейв»** спасает круг, когда запахло аннуляцией. Можно 2-й круг на тёплом → in-lap; «прервать» в
любой момент.

Следишь за **тайминг-башней** с границей вылета. Четыре механики живого менеджмента: **эволюция
трассы** (резинится по ходу, поздние круги быстрее — но риск не успеть; ~1.3с за сессию), **лимит
софта** (~3 свежих комплекта), **трафик** (выезд в толпу компрометирует круг), **флаги** (красный
замораживает сессию и аннулирует круги — «банкер» спасает от вылета). 20 болидов грида хост
симулирует сам (ИИ тоже валит секторные круги под риском, в крайнем случае едет на «сейве» ради
времени); каждый игрок — своя машина, вылетевший наблюдает. **Десктоп-раскладка:** тайминг-башня
слева, карточка живого круга справа (дельты по секторам, контрол пуша, выезд). Сегменты стартуют на
паузе (▶). Итог сегментов → стартовый грид → гонка. Сетап заблокирован (parc fermé).

## Редактор трассы

`localhost:8000/editor.html` — конструктор трасс. Двигай опорные точки (дорога перерисовывается
живьём, как в игре), ставь объекты. **Колесо** — зум к курсору, **средняя кнопка** — пан, **⊡ по
размеру** — сброс вида. Режимы: **Точки** (форма + объекты), **Пит** (поставить боксы + потеря в
питах + рисуешь пит-лейн (вход/выход/сторона/ширина), по которому машины реально едут в гонке (2D+3D)), **Зоны** (на сетке 18 мини-секторов: красишь зоны обгона тормозн./слип + ease; ПКМ по
сектору — класс поворота · 🎯 Авто — расставить зоны по геометрии (правишь после)), **Старт**
(клик по трассе переносит линию старт/финиш в ближайшую опорную точку; «↺ Развернуть» — сменить
направление круга). **Сохранить** → трасса открывается в 3D-гонке; Экспорт/Импорт JSON.

**Пак трасс (общий через гит):** запусти редактор через `node tools/editor_server.mjs` — тогда
**Сохранить** пишет трассу файлом в `ApexWeb/tracks/<slug>.json` (и в localStorage), а **📤
Опубликовать** делает `git add ApexWeb/tracks && commit && push`. После пуша трасса грузится у всех
из папки `tracks/` (в редакторе — группа «Из репо», гонять — через 🏁). Без node-сервера Сохранить
остаётся только в localStorage.

**Превью рядом:** сверху панели — **3D-вид** (тот же движок, что в игре: трасса, сценерий, объекты и
пара машинок едут по линии; **🔄 Обновить 3D** перестраивает его под правки, мышь — орбита/зум) и
**svg-эталон** (оригинальный контур выбранной трассы для сравнения). 3D грузит THREE с CDN —
без сети превью отвалится с тостом, а 2D-редактор продолжит работать.

**🏁 Гонять (сим)** — гонка прямо на твоей трассе: её повороты/прямые определяют, где случаются
обгоны (слипстрим/грязный воздух + твои зоны), пит-потеря = твоё значение. Базовый темп круга пока
от Барселоны (геометрия влияет на бой, не на чистый темп — это следующий шаг).

## 3D-вид гонки (broadcast)

Полноэкранный 3D-вид (кнопка из гоночного HUD) переработан в духе Motorsport Manager — вся гонка
рисуется как телетрансляция:

- **Покрытие и поребрики.** Дорога — крашеная текстура (асфальт, кромки, разметка) с **гравийными
  ловушками и асфальтовыми зонами вылета снаружи поворотов**; поверх неё — **объёмные 3D-поребрики**
  (красно-белые, инстансингом) по обеим кромкам в поворотах. То же покрытие видит редактор трасс.
- **Окружение трассы** (детерминированно по имени трассы, модуль `scenery.js`): **трибуны с толпой**
  вдоль длинных прямых, **шинные барьеры** по внешней стороне поворотов, **рекламные щиты**,
  фоновые **деревья**, **посты маршалов** и **стартовая арка** над линией старт/финиш. Повторяющиеся
  объекты — через `InstancedMesh` (дёшево для отрисовки).
- **Болиды** — обновлённый силуэт F1: нос с законцовкой, понтоны, воздухозаборник, антикрылья,
  **halo**, шлем пилота в цвет команды, шины; кузов в цвете команды с лёгким блеском. **Боковина
  шин окрашена по компаунду** (soft красный / medium жёлтый / hard белый / inter зелёный / wet
  синий) и обновляется вживую при смене резины — стратегия читается прямо на трассе.
- **Камера: 3 режима** (кнопка «Камера»): **ТВ-режиссёр** (сам выбирает самую плотную дуэль на
  трассе и показывает её с разных трасс-камер, перекадровка каждые ~4–6 с), **обзор** (медленный
  облёт всей трассы + драг/зум), **погоня** за своей машиной / лидером с предиктивным взглядом вперёд.
- **Свет и атмосфера** — тёплый «золотой час» key-light + холодный fill, длинные тени, дымка-туман
  для глубины, ACES-тонмаппинг и мягкий bloom (с graceful-фолбэком, если CDN-постобработка не
  загрузилась). Всё — render-only поверх детерминированного сима: геймплей и нетворкинг не затронуты.
- **Погода в 3D** (по `snapshot.wetness` из сима): мокрый асфальт **блестит** (роняем roughness,
  поднимаем metalness → блики от солнца), небо/туман **сереют**, солнце приглушается, идёт **дождь**
  (частицы), а из-под колёс летят **брызги** (полупрозрачный шлейф, интенсивность по влажности).
  Сухо ↔ дождь меняются плавно вслед за погодной моделью сима.

- **Паритет с MM (2026-06):** над каждым болидом — **плавающая бирка** «P{позиция} {код пилота}»
  в цвете команды (спрайт-биллборд, обновляется вживую при смене позиции); на полотне — **чёткие
  белые кромки** и **«резиновая» гоночная линия** по центру; старт-финиш **обжит** (пит-билдинг с
  боксами + плотные трибуны); модели болидов **сглажены** (капсульные корпус/понтоны + конусный нос
  вместо «коробок»); масштаб трассы увеличен (WORLD 175) — круг крупнее, машины читаются мельче.
- **Болид-модель v2 + анти-наложение (2026-06):** болид собран как узнаваемый F1 — скруглённый
  корпус-капсула, конусный нос, **многоэлементные** переднее/заднее крылья с законцовками, кокпит
  с halo и шлемом, хромированные ступицы, цветные обода шин; кузов с **глянцевым лаком**
  (`MeshPhysicalMaterial` clearcoat). В кадровом цикле — **боковое расталкивание** (render-only):
  близкие по дистанции болиды разъезжаются по полотну и едут бок-о-бок, а не сквозь друг друга
  (плюс убирает дёрганье от «толкотни»); постановка сглажена двойным лоу-пассом.

**Быстрый просмотр без уикенда:** открой `http://localhost:8000/showroom.html` — автономный
**3D-шоурум**: полный грид из 22 машин едет по сцене, переключение камер/трасс «на лету» и кнопка
дождя.
