# ApexWeb — дизайн-система («Бродкаст»)

Единый визуальный язык, чтобы новые фичи не расходились. Решения приняты 2026-06 после того,
как фич стало много, а панели начали писаться инлайн-стилями с хардкод-цветами.

## Направление: «Бродкаст» (ТВ-графика F1)

Тёмная база (как было) + яркие акценты: командный/семантический **цвет-спайн** слева у карточек,
крупные жирные числа, **скошенные чипы** (`skewX(-10deg)`), повышенный контраст на ключевых
сущностях. Спокойный «пит-уолл» остаётся фундаментом разметки, «бродкаст» добавляет энергию.

## Информационная архитектура (2 уровня)

Вместо 9 плоских вкладок — группы с под-вкладками (`season.js`, `GROUPS`):

- **Обзор** — дашборд, сезонная история, новости
- **Уикенд** — Календарь (+ карточка след. этапа с целями спонсоров и прогнозом погоды в футере)
- **Гараж** — Машина · **Пит-крю** · Штаб
- **Состав** — Пилоты · Академия · Трансферы
- **Финансы**
- **Зачёт**

Группбар (`.pad-group`/`.pad-gtab`, подчёркивание цветом `--bc`) + сабтаб-бар (`.pad-subtabs`)
только когда в группе >1 вкладки. Состояние — `ctx._padTab`; активная группа выводится из него.

## Токены (style.css `:root`) — НЕ хардкодить хекс в панелях

- Базовые: `--bg --panel --content2/3 --ink --muted --border --accent --good --warn --bad --info`.
- Бродкаст: `--bc` (циан #00b3ff), `--accent2` (тил #00e0b0).
- Домен: компаунды `--tyre-{soft,medium,hard,inter,wet}`, серии `--series-{f4,f3,f2,f1}`.
- Радиусы `--r-sm/md/lg`, тени, `--ease`.

## Кит компонентов (`src/ui/kit.js`)

Чистые строковые билдеры — переиспользовать вместо ручной разметки:

- `bcard({title, chip, spine, body})` — карточка со спайном + uppercase-заголовком + чипом.
- `bchip(text, bg)` — скошенный чип (текст контр-скошен внутри).
- `bbar(pct, color, h)` / `meterRow(name, pct, color, value)` — метр-бар и метр-строка.
- `bkpi(label, value, color)` / `bkpiStrip(items)` — компактные KPI.
- Семантика: `TYRE_COL`, `SERIES_COL`, `moraleCol(m)`, `fatigueCol(f)`.

**Эталон стиля:** панель «Пит-крю» (`season.js`) собрана целиком на ките — на неё ориентироваться
при переводе остальных вкладок.

## Статус миграции

**Сделано:** токены + кит + 2-уровневые вкладки + пит-крю как эталон + **весь паддок переведён**:
все хардкод-хексы в `season.js`/`teamviz.js` заменены на семантические токены (amber→`--warn`,
blue→`--info`/`--series-f3`, green→`--good`, red→`--bad`, серии→`--series-*`), и все панели-обёртки
стали `.bcard` со спайном (`--spine`: команда у Пилотов/Пит-крю/героя, `--bc` у вторичных) и
uppercase-заголовком `.bcard-title`. Спец-панели событий (выкуп/предложение со-директора) сознательно
оставлены на `.panel` как акцентные.

## UX-редизайн: разработка «по направлениям»

Проблема старого экрана «Машина»: 3 под-вкладки × 6 деталей × 3 размера × 3 подхода, без подсказок
и без сравнения с полем — игрок не понимал, что развивать. Решение (модель в `development.js`,
UI в `season.js`):

- **5 направлений** (`DEV_AREAS`) вместо деталей: Аэро/Резина/Эффективность/Надёжность + Мощность ДВС.
- **Ты vs Поле** (`fieldAvg`) — у каждого направления полоса твоего показателя и маркер среднего по
  гриду; слабейшее помечается «СЛАБО».
- **Совет инженеров** — одна строка с рекомендацией развивать слабое место.
- **Одна интенсивность** (`INTENSITY`: Осторожно/Стандарт/Ва-банк) вместо отдельных размер+подход.
- **Автоподбор детали** (`bestPartForArea`) — лучшая деталь под направление считается за игрока.
- Переключатель **«по деталям»** сохраняет старый микроменеджмент для желающих.
- ДВС — отдельная вкладка (токены/вне капа). Команда `career_project` переиспользуется без изменений;
  глубокая модель (потолок/исходы/обкатка) под капотом не тронута. Проверка: `rnd_check` 14/14.
