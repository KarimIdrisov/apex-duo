# Apex Web — Финансы, владение командой и ДВС-программы (дизайн-док)

Статус: черновик на сверку (2026-06). Консолидирует обсуждение: реализм финансов F1,
архетипы финансирования команд, программы силовой установки (ДВС), систему событий
(выкупы/кризисы/живой грид/денежные события) и балансные правки (реактивный ИИ, потолок
скилла). Цель — свериться до написания кода.

## 0. Принципы (не нарушать)

- **Честно и реалистично.** Призовые в F1 почти плоские (анти-отрыв) — копируем. ИИ не читерит
  (сложность масштабирует собственный темп/решения ИИ, без «резинки»).
- **Осмысленный выбор команды.** Заводская = мощь+стабильность+давление; независимая =
  гибкость+риск+скрап. Разные «ощущения» прохождения.
- **Детерминизм.** Все события и развитие — по сиду (воспроизводимо). Два RNG-потока как в симе.
- **Изоляция симуляции.** Любое мета→сим влияние идёт ТОЛЬКО через composed-индикаторы машины
  (`effectiveCar` → power/aero/tyre/fuel/rel) и атрибуты пилота. Меню денег/ДВС не трогают `sim.js`.
- **Кооп-безопасно.** Все решения — кооп-команды (хост применяет, рассылает); крупные выборы —
  общие для со-директоров.

## 1. Реальная F1 — что моделируем (источники в README/чате)

- **Выплаты F1 (Договор Согласия), пул ~$1.27 млрд/год:** ~75% по прошлогоднему месту в Кубке,
  ~20% за историю (топ-3 за 10 лет), 5–10% «наследие» Ferrari. **Чемпион ~$140M, последний ~$60M
  → всего ~2.3×** (намеренно плоско).
- **Спонсоры:** ~$2.05 млрд на 10 команд (~$200M в среднем), титульные до ~$100M/год.
- **Заводские vs клиентские:** концерн финансирует заводскую и **разрабатывает её ДВС вне
  кост-капа** ($50–100M/год скрытого преимущества). Клиентские покупают ДВС (фикс. плата, чуть
  урезанная спека), живут на призовые+спонсоры+вливания владельца.
- **Кост-кап ~$140M** НЕ включает зарплаты пилотов, разработку ДВС, маркетинг → реальные бюджеты ~$200M+.

## 2. Архетипы финансирования команды

Новое поле на каждой команде (`data.js TEAMS[].backer`):

```
backer: {
  type: "works" | "independent",
  grant: $k/сезон,            // годовой грант от концерна/владельца (пол дохода)
  puMaker: bool,              // есть ли своя силовая установка (разработка ДВС)
  supplier: "Mercedes"|...|null, // у кого берём ДВС (если не puMaker)
  boardProfile: "oem" | "owner" | "patient", // характер совета
}
```

- **Заводская (works):** крупный `grant` (стабильный доход), `puMaker:true` (см. §4, ДВС вне
  кост-капа), требовательный совет (`oem`): высокая планка, **риск ухода концерна** при провале (§5).
- **Независимая (independent):** малый/нулевой `grant`, `puMaker:false` (клиентский ДВС, плата за
  поставку), рычаг **разового вливания владельца**, более терпеливый, но бедный совет.

Стартовая раскладка грида 2026 (примерно): Mercedes/Ferrari/Red Bull/Audi(Sauber)/Alpine/Aston —
заводские; Williams/Haas/RB/Cadillac — независимые/клиентские (точные значения — таблица в §7).

## 3. Призовые и экономика (ребаланс под реальность)

- **Сплющить сезонный призовой фонд** до ~2.3× (сейчас 6.2×): напр. P1 ~$16M … P11 ~$7M.
  Формула: `fund = base · (0.43 + 0.57·(N−pos)/(N−1))`, base≈16M, N=11. Анти-отрыв = честнее.
- **Призовые за гонку** оставить (хороший пер-рейс прокси).
- **Кост-кап под реальность:** в кэп идут разработка шасси + персонал + трансферы; НЕ идут зарплаты
  пилотов, **разработка ДВС (у заводских — за счёт концерна, вне кэпа)**, маркетинг/спонсоры.
- **Сверка чисел (цель):** грант малых + плоские призовые + кост-кап должны позволять андердогу
  пробиться за ~2–3 сезона; топ не должен конвертировать финансовый отрыв в недосягаемую машину
  (кэп ограничивает шасси-траты; ДВС-преимущество заводских — отдельная, осознанная асимметрия).

## 4. Программы силовой установки (ДВС) — новая механика

Разделяем разработку на **шасси** (как сейчас, PARTS aero/tyre/fuel...) и **ДВС**.

### 4a. Заводская команда — меню разработки ДВС
- Отдельная вкладка «Силовая установка» (рядом с «Машина»). Три ветки:
  **Мощность (ДВС)**, **Энергия/ERS**, **Надёжность**. Развитие как у шасси-деталей (уровни,
  проекты, стоимость, гонки, риск), но:
  - **вне кост-капа** (финансирует концерн) — реалистичная асимметрия заводских;
  - композится в сим через `power`/`energy`/`rel` (сим уже читает их), отдельно от шасси-деталей.
- **Поставка ДВС клиентам:** заводская команда снабжает N клиентов → **доход с поставки**
  (supply fee) + размазывает стоимость разработки. (Уже есть в Godot-версии: Ferrari → Ferrari/Haas/...)

### 4b. Клиентская команда — путь к своему ДВС
- Сейчас: платит supply fee, получает **неразвиваемый, чуть урезанный** ДВС поставщика.
- **Амбиция «свой ДВС»:** многосезонный проект — найти/заключить **сделку с крупным автоконцерном**
  (через меню/событие §5): крупные деньги + несколько сезонов на постройку PU-программы → выпуск
  собственного развиваемого ДВС (становишься `puMaker:true`). Варианты:
  - **полноценная программа** (как Audi/Red Bull Powertrains): дорого и долго, но свой ДВС;
  - **бейдж-партнёрство** (как Aston-Honda): быстрее/дешевле, ДВС концерна под твоим именем,
    ограниченное влияние на разработку.
- Карьерная арка: скрап-клиент → сделка с концерном → заводская PU-команда.

### 4c. Сеть поставок ДВС
- Поле `supplier` + «кто кого питает». Смена поставщика в межсезонье = **интеграционный штраф**
  (−% эффекта на 1–2 этапа). Концерн может урезать поставку клиенту-сопернику (политика).

## 5. Система событий (выкупы / кризисы / живой грид / деньги)

Детерминированно по сиду; редкие, высоковесные, **завязаны на результат**, не спам.
Проверка на границе сезона (+ малая вероятность в межэтапье).

### 5a. Предложения выкупить твою команду (суиторы)
- Типы: **автоконцерн** (делает заводской: грант + ДВС вне кэпа, требовательный совет),
  **техногигант/конгломерат** (огромный кэш + титульный спонсор, остаёшься на клиентском ДВС),
  **суверенный фонд/миллиардер** (бездонный бюджет + амбиции, нетерпеливый совет).
- **Интерес ∝ привлекательности:** восходящая траектория в Кубке, брендовая стоимость (доход
  спонсоров), текущее место, и **независимость** (концерн скупает перспективных независимых).
- Оффер: разовый кэш + новый годовой грант, новая (часто выше) планка совета, иногда условия
  (их ДВС/их юниор). **Идентичность — на выбор игрока:** оставить имя/ливрею ИЛИ ребренд
  (новое имя+цвета). Отказ — сохраняешь независимость; суитор может вернуться/уйти.

### 5b. Кризис: концерн уходит
- Заводская команда, проваливающая сезон-два → совет концерна **срезает грант/уходит** → возврат
  в независимые с кризисом финансирования (история Honda/BMW/Toyota). Пара к давлению `oem`-совета.

### 5c. Живой грид (соперники)
- Независимого соперника тоже **выкупают**: становится заводским и усиливается (грант/ДВС). Грид
  эволюционирует между сезонами — у топов меняются бекеры, появляются новые заводские силы.

### 5d. Денежные события (мелкие разовые)
- Вливание владельца, внезапный крупный спонсор, штраф FIA, бонус за веху. Небольшие, добавляют
  вариативности; детерминированы.

## 6. Баланс гонки и карьеры (параллельный трек)

- **Реактивный ИИ** (`ai_strategy.js`): на высокой сложности соперник **прикрывает андеркат**
  (упреждающий ответный пит при угрозе) и **активнее защищается**; на лёгкой — нет. Честно
  (масштаб от difficulty), без «резинки».
- **Ошибки ИИ на низкой сложности:** редкие настоящие бландеры (запоздалый/лишний пит, не та резина
  в переменке) — низкая сложность обыгрывается смекалкой.
- **Потолок скилла игрока:** замерить через `tools/balance.mjs`, насколько сетап+квала+стратегия
  «перевозят» машину; цель — уверенно бить ~1 тир выше, иногда красть результат на 2 тира через
  SC/погоду/стратегию, но не доминировать на плохой машине. Подкрутить величину setup-бонуса/плечо.
- **Не трогать** аудит-коридоры сима (сходы/разброс/обгоны/андеркат) — проверять после правок.

## 7. Изменения модели данных

- `data.js TEAMS[]`: добавить `backer{type,grant,puMaker,supplier,boardProfile}`; имя/цвет —
  редактируемые (для ребренда).
- `career`: `puParts` (если puMaker), `puProject`, `puProgram` (если строим свой ДВС),
  `backer`-снимок, `eventsQueue`/`acquisitionOffer`, `parentMood`, `identity{name,color}`.
- Поднять `CAREER_V`, добавить миграцию (старые сейвы получают дефолтные бекеры/ДВС по таблице).
- Все новые числа — `const`-таблицы вверху модулей (баланс правится там).

## 8. Поэтапный план (каждая фаза: детерминизм + кооп + изоляция сима + тесты/коридоры + README)

- **Фаза A — финреализм:** архетипы бекеров (works/independent + грант), плоские призовые,
  кост-кап под реальность (ДВС вне кэпа у заводских). Фундамент.
- **Фаза B — ДВС-программы:** меню разработки ДВС для заводских (вне кэпа) + путь клиента к своему
  ДВС (сделка+постройка) + экономика поставок.
- **Фаза C — события:** суиторы/выкупы (идентичность на выбор) + уход концерна + живой грид +
  денежные события.
- **Фаза D — баланс:** реактивный ИИ (± ошибки на низкой сложности) + замер/тюнинг потолка скилла.

Порядок можно менять; A — основа для B и C.
